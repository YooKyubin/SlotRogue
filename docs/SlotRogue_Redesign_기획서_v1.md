# SlotRogue 리디자인 기획서 v1

> 작성: 시니어 게임 기획 관점 / 대상: SlotRogue 무한모드(v1) + 아웃게임 메타
> 전제: 기존 코드 자산(심볼 풀, 패턴 카탈로그, 유물 80종, 웨이브 스케줄, Luck 강제족보, 리더보드 훅)을 **재배선**한다. 신규 엔진을 만들지 않는다.

---

## 0. 진단 요약 (왜 바꾸는가)

현재 빌드의 근본 문제는 세 가지가 서로 분리돼 있다는 점이다.

| 문제 | 현상 | 근거 |
|---|---|---|
| 테마 ↔ 메커니즘 단절 | 심볼 종류가 피해에 무관(777=레몬3개) | `SlotResultCalculator` 심볼 미참조 |
| 전투 의사결정 0 | 한 턴 = 스핀 버튼 1회 + RNG 구경 | `SlotTurnController.SpinAsync` |
| 난이도 = HP 스펀지 | 적 공격력 고정, HP만 증가 | `EncounterScaling`이 MaxHp만 스케일 |
| 빌드 다양성 부재 | 작동 유물 ~13/80, 전부 평평한 스탯 | Phase1 정책 |
| 보상 희박 | 10전투 중 8번은 +4 회복뿐 | `GameFlowSession.CompleteInfiniteVictory` |
| 메타 부재 | 항상 패배로 끝나는데 재방문 동기 없음 | 칭호/도감/리더보드 미구현 |

**핵심 진단:** 지금 필요한 건 "튜닝"이 아니라 **코어 루프 재설계**다.

## 1. 디자인 비전 & 핵심 기둥

> **한 줄 비전:** "슬롯을 굴려 족보를 만들고, *심볼 정체성 × 유물 시너지*로 빌드를 쌓아 더 깊이 들어가는 푸시-유어-럭 로그라이크."

세 개의 기둥:

1. **심볼은 '무엇으로 변환되는가', 족보는 '얼마나 큰가'** — 둘을 곱해 한 턴의 결과가 나온다. (테마↔메커니즘 결합)
2. **모든 턴에 푸시-유어-럭 결정 1개** — Luck 자원으로 홀드/재스핀/등급 상향. (의사결정 밀도)
3. **달릴수록 강해지고, 죽어도 남는다** — 인런 빌드(휘발) + 아웃게임 해금(영구). (리텐션)

## 2. 인게임 재설계

### 2.1 심볼 정체성 — 변환(Conversion) 시스템 ★핵심

기존 패턴 가치 `V = 칸 수 × 족보 배율`은 **그대로 유지**한다. 바뀌는 것은 "V를 무엇으로 바꾸는가"이며, 이를 **매칭된 심볼**이 결정한다.

| 심볼 | 확률군 | 정체성 | 변환 규칙(예시) |
|---|---|---|---|
| 체리 Cherry | 고(8) | 다단 안정딜 | 피해 = V×1, **공격횟수 = 칸 수** (다단 유물 시너지) |
| 종 Bell | 고(8) | 자원 | 피해 = V×1.5 + **골드/별조각 생성** |
| 클로버 Clover | 고(8) | 행운 | **Luck +V** (+ 소량 피해 V×1) |
| 7 Seven | 저(4) | 폭딜 | 피해 = **V×3** (단일 고배율) |
| 다이아 Diamond | 저(4) | 방어/반격 | **방어 = V×2** (반격 빌드의 베이스) |
| 레몬 Lemon | 저(4) | 상태이상/와일드 | **중독 등 스택 부여** + 와일드(인접 매칭 보조) |

설계 의도:
- 고확률군 = **꾸준한 가치**(딜/자원/행운), 저확률군 = **스파이크**(폭딜/방어/상태). 이는 기존 풀 주석의 "고확률=낮은 수치, 저확률=높은 수치" 철학과 정합.
- 가방 구성이 곧 빌드 정체성이 된다: "7을 쟁여 한 방 노림(저명중·고폭발)" vs "체리로 다단 안정" vs "레몬 중독 도트".
- 구현: `SlotResultCalculator`가 패턴별 `Symbol`을 읽어 변환 테이블을 타게 한다. 패턴 가치 계산부는 재사용.

> **마이그레이션 노트:** 변환 테이블은 ScriptableObject(`SymbolConversionTable`)로 빼서 밸런스 핫픽스 가능하게. 유물은 이 변환을 *증폭/치환*(예: "체리 변환 시 피해 +50%", "클로버를 7로 취급")한다 → 심볼=베이스, 유물=빌드.

### 2.2 턴 의사결정 — Luck 자원 + 홀드/재스핀 ★핵심

매 턴 단 하나라도 능동 결정을 강제한다. 신규 자원 **Luck**(클로버로 충전)을 도입하고, 스핀 결과를 본 뒤 다음 중 택1:

- **확정(Resolve):** 그대로 족보 적용. (Luck 0)
- **홀드 재스핀(Hold & Respin):** 셀 최대 N개 잠그고 나머지 재추첨. (Luck 비용)
- **등급 상향(Force Up):** Luck를 써서 이번 판에 강제 족보를 1개 끼워 넣음 — **이미 구현된 `SlotPatternCatalogAsset.PickForcedPattern(random, luck)`을 그대로 활용.**

이렇게 하면 클로버(Luck 생성) → 재스핀/상향(Luck 소비) → 7/대형족보 추격이라는 **자기완결 루프**가 생긴다. 클로버가 "그냥 고확률 꽝 심볼"에서 빌드 엔진으로 승격된다.

### 2.3 족보/계산 모델 개편

- **대형 족보 도달성 복구:** 대지/천상/눈/잭팟은 "지정 8~15칸 *동일 심볼*" 조건이라 가중 가방에서 사실상 불가능 → **"라인/형태 완성"(같은 심볼 불요, 모양만)** 으로 완화하거나, 도달은 홀드·와일드(레몬)·강제상향으로만 가능한 **빌드 보상**으로 재배치. 잭팟이 영원히 안 터지는 슬롯은 금지.
- **가독성:** 패턴 중복 억제(`SlotPatternResolver`)가 왜 일어났는지 셀 하이라이트/툴팁으로 시각화. "이 라인은 더 큰 족보에 흡수됨" 명시.

### 2.4 유물 시너지 & 활성화 로드맵

현재 80종 중 ~13종만 활성(상태이상/배율패시브/전투훅 유물은 전투 코어 불일치로 비활성). 이를 **버그가 아니라 진행 콘텐츠로 전환**한다:

- 전투 코어 정합 작업으로 카테고리를 순차 개방(아래 로드맵 §7).
- 미개방 유물은 아웃게임 **해금 트리**에 묶어 "플레이할수록 풀이 넓어지는" 메타 보상으로 활용 → 동질 콘텐츠 문제와 콘텐츠 게이팅을 동시에 해결.

필수 코어 정합 항목(상태이상): 화상(즉시+몬스터턴 유지), 감염=중독(턴종료 피해 후 1감소), 취약/약화/가시. 이것이 풀리면 연금술/반격 빌드가 살아난다.

### 2.5 난이도 스케일링 재설계

HP 스펀지 금지. 다축 스케일:

| 축 | 현재 | 변경 |
|---|---|---|
| 적 HP | 전투/섹션 선형↑ | 유지(완만화) |
| 적 공격력 | **고정** | 전투/섹션 스케일 추가 (`EncounterScaling`에 DamageScale 도입) |
| 적 행동 | 고정 패턴 | 섹션 진행 시 상태이상/방어/소환 패턴 해금 |
| 군집 | 무한모드 단일 적 | 섹션 2+부터 다중 적(이미 있는 다중 로스터/진형/타겟 시스템 활용) |
| 엘리트/보스 | HP 배수만 | 고유 기믹(접두사 어픽스, 페이즈) |

→ "잉여 복잡도"였던 다중 적/타겟/진형이 **후반 난이도 장치로 회수**된다.

### 2.6 보상 케이던스 — 전투마다 드래프트

"8연속 무선택" 제거. **모든 전투 승리 후 3택1 드래프트**:

- 일반 전투: [심볼 추가 / 마이너 유물 / 회복+Luck] 중 3장 제시, 1택. (8/+4 회복은 한 옵션으로 흡수)
- 엘리트/보스: **유물 3택1**(레어+ 포함) + 옵션 "심볼 1개 정화(제거)" — 가방 압축 결정.

기존 `RunRewardType` + 유물 보상 UI를 드래프트 프레임으로 재사용.

## 3. 아웃게임(메타) 설계 ★신규

무한모드는 결국 패배로 끝난다. **"기록을 남기고, 다음 런을 더 강하게 시작"** 하는 메타가 리텐션의 전부다.

### 3.1 로비 허브

로비를 단순 Play 버튼에서 **허브**로:

```
[로비]
 ├─ 플레이 (이어하기 / 새 런 / 캐릭터 선택)
 ├─ 해금 상점 (별조각 소비)
 ├─ 캐릭터(아키타입) 선택·장착
 ├─ 도전과제 / 칭호 장착
 ├─ 도감 (심볼·유물·몬스터)
 ├─ 리더보드 (주간 / 승천별 / 일일도전)
 └─ 일일·주간 도전
```

### 3.2 메타 통화 & 영구 해금

- **별조각(Stardust):** 영구 통화. 획득 = `f(도달 전투수, 처치 엘리트/보스, 달성 도전과제)`. 런 종료 시 정산.
- **골드:** 인런 통화(종 심볼 생성). 드래프트 리롤/정화 비용 등에 소비. 런 종료 시 소멸.
- **해금 상점(별조각 소비):**
  - 신규 유물을 **활성 풀에 추가** (미개방 67종을 단계적으로) — 콘텐츠 게이팅을 진행 보상으로.
  - 신규 캐릭터/시작 유물 해금.
  - **권장: 영구 스탯 버프는 최소화.** StS 철학대로 "파워"보다 "다양성"을 판다. 메타로 인런을 trivial하게 만들면 안 된다.

### 3.3 캐릭터(아키타입) — 출시 3종

각 캐릭터 = 시작 가방 비율(`SlotSymbolPool` 프리셋) + 시작 유물 + 패시브. 리플레이 다양성의 핵심.

| 캐릭터 | 시작 비중 | 패시브 | 빌드 방향 |
|---|---|---|---|
| 도박사 Gambler | 7↑ | 시작 Luck 보유, 7 변환 폭딜↑ | 하이리스크 폭딜 |
| 연금술사 Alchemist | 레몬↑ | 중독 스택 강화 | 상태이상 도트 |
| 수문장 Warden | 다이아↑ | 방어의 X%만큼 반격 | 방어→반격 |

(추가 캐릭터는 해금 상점/도전과제 보상으로 확장.)

### 3.4 도전과제 · 칭호 · 도감 · 리더보드

- **도전과제 → 칭호:** 도전과제 보상으로 칭호 획득(예: 잭팟 1회=[한탕], 상태이상만으로 보스 처치=[연금술사], 무피해 보스=[철벽]). 로비에서 장착, **리더보드에 표시** — 이미 준비된 `LeaderboardView` DesignationText 훅 사용.
- **도감:** 조우한 심볼/유물/몬스터 기록. 완성도 보상(별조각/칭호). 수집 동기.
- **리더보드:** 주간 랭킹(이미 `WeeklyRankingSchedule` 존재) + 승천 등급별 + 일일 도전별. 칭호로 자기표현.

### 3.5 난이도 승천(Ascension) & 일일·주간 도전

- **승천 1~20:** 마일스톤(보스 첫 처치 등) 달성 시 해금. 각 단계가 모디파이어 추가(적 HP/공격↑, 회복↓, 저주 유물 등장률↑, 시작 Luck↓ 등) + **점수 배율↑**. 장기 리텐션과 리더보드 공정성(같은 승천 내 경쟁)을 동시 확보.
- **일일/주간 도전:** 고정 시드(이미 `RunSeed` 존재)로 전원 동일 보드 → 순수 실력 비교 리더보드.

## 4. 모드 구조 & 진행 플로우

```
[타이틀]
   ↓
[로비 허브] ←──────────── 런 종료(별조각 정산, 도전과제/도감 갱신)
   ↓ (캐릭터 선택 → 새 런 / 이어하기)
[무한 런]
   섹션1(전투1~10)  → 보스
   섹션2(전투11~20) → 다중 적·상태이상 적 해금, 보스
   섹션3+ ...        → 어픽스/페이즈 보스
   각 전투: 스핀 → (Luck 결정) → 족보 변환 → 전투 → 승리 드래프트(3택1)
   패배: GameOver(도달 기록·별조각 정산) → 로비
```

승천·일일도전은 무한 런 위에 모디파이어/시드를 얹는 변주.

## 5. 경제 밸런스 프레임 (초기값, 핫픽스 대상)

**인런 — Luck**

| 항목 | 값(예시) |
|---|---|
| 클로버 변환 | Luck += V |
| 홀드 재스핀 | Luck −3 / 회 |
| 강제 등급 상향 | Luck −5 |
| 시작 Luck(도박사) | 5 |

**인런 — 회복/딜 (스폰지 방지 원칙)**

- 적 HP 성장 slope < (심볼변환 평균딜 + 유물 + 드래프트) 성장 slope 가 되도록 검증.
- 적 공격력도 스케일하므로 회복 +4/전투 같은 무지성 회복은 폐지 → 드래프트 옵션으로만.

**아웃게임 — 별조각**

| 행동 | 별조각 |
|---|---|
| 일반 전투 클리어 | 1 |
| 엘리트/보스 | 5 / 10 |
| 신규 도전과제 | 가변 |
| 승천 단계 배율 | ×(1+0.1·승천) |

(실수치는 플레이테스트로 확정 — 본 문서는 구조와 시작점만 고정.)

## 6. 개발 로드맵 & 기존 코드 매핑

### Phase 1 — 코어 루프 회복 (출시 필수)
1. **심볼 변환 시스템** — `SymbolConversionTable`(SO) + `SlotResultCalculator` 심볼 참조.
2. **Luck 자원 + 홀드/강제상향** — `PickForcedPattern` 연결, 턴 UI.
3. **드래프트 보상(전투마다 3택1)** — `RunRewardType`+유물 UI 재사용.
4. **난이도 다축 스케일** — `EncounterScaling`에 DamageScale 추가.
5. **로비 허브 + 별조각 + 캐릭터 3종** — `GameFlowSession`/로비 확장.

### Phase 2 — 빌드 깊이 & 메타 확장
6. **전투 코어 상태이상 정합** → 상태이상/반격/패시브 유물 카테고리 개방.
7. **해금 트리(유물·캐릭터)** — 별조각 소비.
8. **도전과제 → 칭호 → 리더보드 표시** — DesignationText 훅 완성.
9. **다중 적/어픽스 보스** — 기존 다중 로스터/진형/타겟 회수.

### Phase 3 — 장기 리텐션 & 소셜
10. **승천 1~20**, **일일/주간 도전**(RunSeed), **도감 완성도 보상**.
11. (반응 양호 시) **스토리모드** 부활 — 동결해 둔 맵 자산 재연결.

## 7. 리스크 & 검증 지표(KPI)

| 리스크 | 완화 |
|---|---|
| 변환 시스템이 기존 밸런스 전면 재작업 유발 | 변환 테이블 SO화로 격리, 패턴 가치부는 재사용 |
| 메타 파워가 인런을 trivial화 | 메타는 "다양성" 판매, "파워" 최소 |
| Luck 루프가 턴을 늘려 템포 저하 | 재스핀 횟수 상한 + 빠른 연출 |
| 상태이상 코어 정합 비용 | Phase 2로 분리, 미개방 유물은 메타 게이팅으로 활용 |

**KPI(검증 목표):**
- 전투당 *의미 있는 결정* ≥ 1 (현재 0)
- 한 런 중앙값 플레이타임 20~30분
- 서로 다른 *유효 빌드 아키타입* ≥ 3 (캐릭터별)
- 첫 런 후 *해금 1개 이상 획득률* ≥ 90% (재방문 동기)
- 대형 족보(잭팟 포함) *런당 최소 1회 도달 가능* (현재 사실상 0)
