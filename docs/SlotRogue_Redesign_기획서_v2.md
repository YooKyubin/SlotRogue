# SlotRogue 리디자인 기획서 v2 — "터지는 슬롯"

> v1에서 바뀐 점: ①심볼은 **중립 유지**(클로버핏 철학) ②피해를 덧셈 → **Base × Mult 곱셈 폭발**(발라트로형)로 전환 ③적 텔레그래프를 **매 턴 목표**로 승격.
> 전제: 기존 코드 자산을 **재배선**한다. 신규 엔진 안 만든다.

---

## 0. 한 줄 비전

> **"엔진을 쌓고 → 레버를 당기고 → 터진다."**
> 한 스핀의 결과 = **Base(족보 가치) × Mult(유물이 쌓은 배수)**. 곱셈이라 빌드가 기대 이상으로 폭발한다.

---

## 1. 핵심 기둥

1. **엔진 → 레버 → 폭발 (재미의 정체).** 결과 = Base × Mult. 곱셈 콤보로 "9 → 90"이 나오는 순간이 도파민. (발라트로)
2. **심볼은 중립, 의미는 유물이 런마다 부여.** 심볼 자체엔 능력 없음. 유물이 "어떤 심볼이 Base/Mult를 주는가"를 정한다 → 조합 유연성. (클로버핏)
3. **적 텔레그래프 = 매 턴 목표.** 예고된 적 행동치를 보고 *버스트 / 방어 / 빌드*를 고른다. (의사결정 밀도 — 이미 있는 인텐트 시스템 활용)
4. **달릴수록 강해지고, 죽어도 남는다.** 인런 빌드(휘발) + 아웃게임 해금(영구). (리텐션 — §3, 단 코어가 재밌어진 *뒤*에)

---

## 2. 인게임 코어 — Base × Mult 점수 모델 ★전부 여기서 갈림

### 2.1 계산식

기존 패턴 가치 `V = 칸 수 × 족보 배율`을 **Base**로 재사용한다. 현재의 `× 2` 플랫 배수는 폐기하고, 그 자리를 **Mult**가 대체한다.

```
Base  = Σ (각 발동 족보의 V)            // 보드가 만든 "원석"
        + Σ (Chip 유물 보너스)          // +N

Mult  = 1
        + Σ (Add-Mult 유물)            // +0.5, +1   (선형 성장)
Mult  = Mult × Π (×Mult 유물)          // ×1.5, ×2   (폭발 — 희귀)

피해   = round(Base × Mult)
```

- **심볼은 Base 계산에 안 들어간다** → 중립 철학 유지 ✔. 심볼은 유물이 "이 매치가 Chip/Mult를 주는지" 판단하는 **후크**일 뿐.
- 초반(유물 적음): Mult ≈ 1.0~1.5 → 선형. 후반(엔진 완성): ×Mult가 곱해지며 ×6, ×10+ → **폭발**.

### 2.2 왜 이게 "재밌는" 곡선인가 — 샘플

**Base 값 예시** (V = 칸수 × 족보배율)

| 족보 | Base | 족보 | Base |
|---|---|---|---|
| 가로3 / 세로 / 대각 | 3 | 지그재그 | 20 |
| 가로4 | 8 | 대지/천상 | ~56 |
| 가로5 | 15 | 잭팟 | 150 |

**같은 보드, 유물 유무 비교**
보드: 체리라인2 + 다이아라인1 → Base 원석 = 3+3+3 = **9**

```
유물 없음:                 9 × 1.0 = 9
엔진 완성:
  [Chip]  매치당 +2 base   → Base 9+6 = 15
  [+Mult] 체리매치당 +0.5  → +1.0
  [+Mult] 3매치↑면 +1      → +1.0
  [×Mult] 다이아 있으면 ×2 → Mult (1+1+1)×2 = 6.0
  = 15 × 6.0 = 90  💥
```

**9 → 90.** 빌드가 원석을 10배로 키웠다. 이 "내 조합이 터졌다"가 한 번 더 돌리게 만든다. 갓롤(×Mult 체인)이면 ×20+ → 보스 삭제 = 스크린샷·자랑각.

### 2.3 유물 = 의미 엔진 (효과를 3버킷으로 재정의)

80종 카탈로그의 효과를 **세 버킷**으로 다시 표현한다. 등급이 곧 폭발력 게이트.

| 버킷 | 등급 경향 | 역할 | 예시 |
|---|---|---|---|
| **Chip (+Base)** | Common | 안정적 바닥 깔기 | "체리 매치당 base +2" |
| **+Mult** | Uncommon | 선형 성장 | "다이아 매치 시 mult +0.5" |
| **×Mult** | Rare/Legendary | **폭발 (빌드 코어)** | "7 매치 시 mult ×2", "매치 5개↑면 ×3" |
| **Enabler** | 전 등급 | 콤보 성립 | "모든 매치를 체리로 취급", "리트리거", "+1 스핀" |

- **심볼-후크가 핵심.** "체리/7/다이아…에 반응"하는 유물(`MatchSymbol`/`MatchTag` 트리거 — *이미 enum 존재*)을 **Phase1 활성 풀에 최우선 투입**. 지금처럼 플랫 +피해 유물만 있으면 심볼은 영원히 무의미하다.
- **시작 유물도 심볼-후크로** → 1턴부터 심볼에 의미가 생긴다.
- **리트리거(재발동)** = 발라트로식 쾌감 폭탄. 족보 1개를 2번 친 것으로 계산. 싸고 강한 도파민.

### 2.4 매 턴 루프 — 텔레그래프를 목표로

```
적 인텐트 표시: "2턴 뒤 12딜"
   ↓
스핀 → 보드 확인
   ↓  ← 푸시-유어-럭 결정 (1개 강제)
   ├ 확정          : 그대로 Base×Mult 적용
   ├ 홀드 재스핀   : 셀 잠그고 재추첨 (턴당 기본 1회, 유물로 추가)
   └ 강제 상향     : 족보 1개 끼워넣기 (PickForcedPattern — 이미 구현, 아이템/유물로 발동)
   ↓
피해 解決 → 적 인텐트와 비교 (버스트로 끝낼까 / 실드로 버틸까)
```

- 재스핀/상향은 **유물·아이템·기본 횟수**에서 나온다. **심볼(클로버)에 능력을 안 박는다** — 중립 철학 일관.
- "다음 적 행동이 보인다"는 이미 있는 장점 → *이번 스핀의 목표치*로 쓴다. 매 턴이 미니 퍼즐이 된다.

### 2.5 연출 = 재미의 절반 (Number Theater)

곱셈 모델은 **연출 없으면 안 터진다.** 필수:

- Base와 Mult를 **분리해 카운트업**: "Base 15 … ×6.0 … = 90!" 숫자 점점 커지고 SFX 고조.
- 유물 기여를 **순차 점등**: Chip 깜빡 → +Mult 더 크게 → **×Mult는 화면 플래시 + 셰이크**.
- 매치 셀 하이라이트로 "왜 이게 카운트됐나" 가독성 확보(중복 억제 포함).

### 2.6 족보 도달성 복구

대지/천상/눈/잭팟이 "지정 칸 *동일 심볼*"이라 사실상 불가능 → **모양만 맞으면 성립**(동일 심볼 불요)으로 완화하거나, 와일드/홀드/강제상향으로만 닿는 **빌드 보상**으로 재배치. 잭팟이 평생 안 터지는 슬롯은 금지.

### 2.7 난이도 = 다축 (HP 스펀지 금지)

적 **공격력·행동(상태이상/방어)·군집(다중 적)** 도 스케일. `EncounterScaling`에 DamageScale 추가. 잉여로 만든 다중 적/타겟/진형을 후반 장치로 회수.
밸런스 가드레일: *좋은 턴 = 일반 적 처치 / 갓 턴 = 원샷*. ×Mult는 Rare+로만 게이트해 초반 폭주 방지.

### 2.8 구현 매핑 (기존 코드 재배선)

| 변경 | 위치 |
|---|---|
| `Base`, `Mult` 필드 추가 | `SlotCalculationResult` / `SlotCombatRequest` |
| Base = Σ패턴가치, Mult=1 산출 | `SlotResultCalculator.Calculate` |
| 유물이 Chip/AddMult/MultMult **기여** 방출 | `RelicTurnResolver` + 유물 효과 모델 |
| `Final = (Base+ΣChip) × ((1+ΣAdd) × Π×Mult)` 조립 | `CombatTurnRequestBuilder.Build` (이미 조립 지점) |
| Base×Mult 카운트업 연출 | 슬롯 프레젠테이션 |

> 메모리상 "슬롯이 패키지 계산 → 전투는 소비만" 구조라 이 변경은 **기존 seam에 그대로 올라간다.**

## 3. 아웃게임(메타) — 코어가 재밌어진 *뒤*

(상세는 v1 문서 §3 유지. 요지만)

- **로비 허브**: 캐릭터 선택 / 해금 상점 / 도전과제·칭호 / 도감 / 리더보드 / 일일도전.
- **별조각(영구 통화)**: 도달 깊이·도전과제로 획득 → **해금 트리**로 미개방 유물 67종 단계 개방(콘텐츠 게이팅 = 진행 보상).
- **캐릭터 3종**: 도박사(×Mult/7) / 연금술사(레몬·상태이상) / 수문장(다이아·반격) — 시작 가방·유물·패시브 차별.
- **도전과제→칭호→리더보드 표시**: `LeaderboardView` DesignationText 훅 사용.
- **승천 1~20 + 일일/주간 도전**: 고정 `RunSeed`로 공정 경쟁 → 매일 들어올 이유.
- **(보류 결정)** 코인 경제+상점(클로버핏형 플레이어 주도 성장)을 넣을지 = 별도 결정. 넣으면 "스핀=코인 획득→상점 구매"로 성장 페이스를 플레이어가 쥔다.

⚠️ **순서 원칙: 재미 먼저, 메타 나중.** 재미없는 게임의 일일 도전은 아무도 안 한다.

## 4. 개발 로드맵

### Phase 1 — "터지는가?" 검증 (최우선)
1. **Base × Mult 점수 모델** (§2.1) — `SlotResultCalculator`/`CombatTurnRequestBuilder` 개편.
2. **심볼-후크 유물 + ×Mult/리트리거** Phase1 풀 투입 (§2.3).
3. **Number Theater 연출** (§2.5) — 이게 없으면 검증 불가.
4. **매 턴 텔레그래프 목표 + 기본 재스핀** (§2.4).
> **검증 프로토타입:** 적 1마리 + 곱셈 유물 3개 쥐여주고 *한 스핀에 피해가 폭발하는지 + "한 번 더" 느낌이 나는지*만 본다. 이 한 판이 코어 증명.

### Phase 2 — 빌드 깊이 & 난이도
5. 유물 80종을 3버킷으로 재정의 + 전투 코어 상태이상 정합(→ 상태/반격 유물 개방).
6. 다축 난이도(§2.7), 보상 드래프트(전투마다 3택1), 족보 도달성(§2.6).

### Phase 3 — 메타 & 리텐션
7. 로비 허브 / 별조각 / 캐릭터 3종 / 해금 트리.
8. 도전과제→칭호→리더보드, 승천, 일일/주간 도전.
9. (반응 양호 시) 스토리모드 부활.

---

## 5. KPI

- 한 판에 **"터졌다" 순간 ≥ 1회** 체감 (정성).
- 후반 갓 빌드 Mult **×10 이상 도달 가능**, 초반 ×Mult 폭주 없음.
- 전투당 *의미 있는 결정* ≥ 1 (텔레그래프 대응).
- 첫 런 후 *해금 1개↑ 획득률* ≥ 90%.
- (메타 진입 후) 일일 도전 *재방문율* — 리텐션 핵심 지표.

---

## 6. 결론

> **Base × Mult 곱셈 + 심볼-후크 유물 + 폭발 연출.** 이 셋이 "재미"의 8할. 나머지(난이도·보상·메타)는 그 위에서 튜닝/확장.
> **첫 삽: Phase 1 검증 프로토타입.** "한 스핀에 터지고, 한 번 더 돌리고 싶다"가 나오면 게임은 산다.
