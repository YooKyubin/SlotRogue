# SlotRogue 리디자인 기획서 (v3 · 통합 최종본)

> 본 문서는 v1·v2 논의를 통합한 최종본이다. 이전 두 문서는 과정 기록으로만 남긴다.
> **두 줄 요약:**
> ① 인게임 — 한 스핀 = **Base × Mult** 곱셈 폭발(발라트로형). 심볼은 중립, 의미는 유물이 런마다 부여(클로버핏형).
> ② 아웃게임 — **모든 죽음이 의미**를 남겨 "한 판 더"를 만든다. 해금은 *파워가 아니라 가능성*을 판다.
> 전제: 기존 코드 자산을 **재배선**한다. 신규 엔진을 만들지 않는다.

---

## 1. 진단 — 왜 바꾸나

| 문제 | 현상 | 근거 |
|---|---|---|
| 폭발이 구조적으로 불가 | 피해 = 패턴가치 × 2 (덧셈·플랫) | `SlotResultCalculator` |
| 심볼이 무의미 | 작동 유물 ~13/80이 전부 플랫 +피해, 심볼-후크 유물 0개 | Phase1 정책 |
| 전투 의사결정 0 | 한 턴 = 스핀 1회 + RNG 구경 | `SlotTurnController.SpinAsync` |
| 난이도 = HP 스펀지 | 적 공격력 고정, HP만 증가 | `EncounterScaling` |
| 보상 희박 | 10전투 중 8번은 +4 회복뿐 | `GameFlowSession` |
| 죽으면 끝 | 재방문 동기 없음, 칭호/도감/리더보드 미구현 | — |

**진단: 튜닝이 아니라 코어 루프 재설계가 필요하다.**

---

## 2. 비전 & 핵심 기둥

> **"엔진을 쌓고 → 레버를 당기고 → 터진다. 죽어도 다음 판이 더 넓어진다."**

1. **엔진 → 레버 → 폭발** — 결과 = Base × Mult. 곱셈으로 "9 → 90"이 나오는 순간이 재미의 정체. (발라트로)
2. **심볼은 중립, 의미는 유물이 런마다 부여** — 심볼 자체엔 능력 없음. 유물이 "어떤 심볼이 Base/Mult를 주는가"를 결정. (클로버핏)
3. **적 텔레그래프 = 매 턴 목표** — 예고된 적 행동치를 보고 버스트/방어/빌드를 고른다. (의사결정)
4. **모든 죽음이 의미** — 별조각·해금·런요약으로 "한 판 더"를 만든다. 단, 코어가 재밌어진 *뒤에*. (리텐션)

# 인게임 설계

## 3. 코어 점수 모델 — Base × Mult

### 3.1 계산식

기존 패턴 가치 `V = 칸 수 × 족보 배율`을 **Base**로 재사용한다. 현재의 `× 2` 플랫 배수는 폐기, 그 자리를 **Mult**가 대체한다.

```
Base  = Σ (각 발동 족보의 V)        + Σ (Chip 유물 +N)
Mult  = 1 + Σ (Add-Mult 유물 +x)               // 선형 성장
Mult  = Mult × Π (×Mult 유물 ×x)               // 폭발 (희귀)
피해   = round(Base × Mult)
```

- **심볼은 Base 계산에 안 들어간다** → 중립 철학 유지. 심볼은 유물이 "이 매치가 Chip/Mult를 주나" 판단하는 후크일 뿐.
- 초반(유물 적음) Mult ≈ 1.0~1.8 → 선형. 후반(엔진 완성) ×Mult가 곱해져 ×6~20+ → 폭발.

### 3.2 Base 값표 & 폭발 샘플

| 족보 | Base | 족보 | Base |
|---|---|---|---|
| 가로3 / 세로 / 대각 | 3 | 지그재그 | 20 |
| 가로4 | 8 | 대지 / 천상 | ~56 |
| 가로5 | 15 | 눈 | ~80 |
| | | 잭팟 | 150 |

**같은 보드(체리라인2 + 다이아라인1 = 원석 Base 9), 유물 유무 비교**

```
유물 없음 :  9 × 1.0 = 9
엔진 완성 :
  [Chip ]  매치당 +2 base       → Base 9 + 6 = 15
  [+Mult]  체리 매치당 +0.5     → +1.0
  [+Mult]  3매치↑면 +1          → +1.0
  [×Mult]  다이아 있으면 ×2     → Mult (1+1+1) × 2 = 6.0
  = 15 × 6.0 = 90  💥
```

**9 → 90.** 빌드가 원석을 10배로. 갓롤(×Mult 체인)이면 ×20+ → 보스 삭제 = 자랑각.

### 3.3 유물 = 의미 엔진 (효과를 3버킷으로 재정의)

80종 카탈로그의 효과를 세 버킷으로 다시 표현한다. 등급이 폭발력 게이트.

| 버킷 | 등급 경향 | 값(초기) | 역할 | 예시 |
|---|---|---|---|---|
| **Chip (+Base)** | Common | +1~3 | 안정적 바닥 | "체리 매치당 base +2" |
| **+Mult** | Uncommon | +0.3~1.0 | 선형 성장 | "다이아 매치 시 mult +0.5" |
| **×Mult** | Rare/전설 | ×1.5~3 | **폭발 (빌드 코어)** | "7 매치 시 mult ×2", "5매치↑면 ×3" |
| **Enabler** | 전 등급 | — | 콤보 성립 | "모든 매치를 체리로 취급", "리트리거", "+1 스핀" |

- **심볼-후크가 핵심.** `MatchSymbol`/`MatchTag` 트리거(이미 enum 존재)를 쓰는 유물을 **Phase1 활성 풀에 최우선 투입.** 이게 없으면 심볼은 영원히 무의미.
- **시작 유물도 심볼-후크** → 1턴부터 심볼에 의미.
- **리트리거(재발동)** = 족보 1개를 2번 친 것으로 계산. 싸고 강한 도파민.

### 3.4 매 턴 루프 — 텔레그래프를 목표로

```
적 인텐트: "2턴 뒤 12딜"
   ↓ 스핀 → 보드 확인
   ↓ ── 푸시-유어-럭 결정 (1개 강제) ──
   ├ 확정        : 그대로 Base×Mult 적용
   ├ 홀드 재스핀 : 셀 잠그고 재추첨 (턴당 기본 1회, 유물로 추가)
   └ 강제 상향   : 족보 1개 끼워넣기 (PickForcedPattern — 이미 구현, 아이템/유물로 발동)
   ↓ 피해 解決 → 인텐트와 비교 (끝낼까 / 버틸까)
```

- 재스핀·상향은 **유물·아이템·기본 횟수**에서 나온다. **심볼(클로버)에 능력을 안 박는다** — 중립 일관.
- 이미 있는 "다음 적 행동 표시"를 *이번 스핀의 목표치*로 승격. 매 턴이 미니 퍼즐.

### 3.5 연출 — Number Theater (재미의 절반)

곱셈 모델은 연출 없으면 안 터진다. **필수:**
- Base와 Mult **분리 카운트업**: "Base 15 … ×6.0 … = 90!" 숫자 커지고 SFX 고조.
- 유물 기여 **순차 점등**: Chip 깜빡 → +Mult 크게 → **×Mult는 화면 플래시 + 셰이크**.
- 매치 셀 하이라이트로 "왜 카운트됐나" 가독성(중복 억제 포함).

### 3.6 족보 도달성 복구

대지/천상/눈/잭팟이 "지정 칸 *동일 심볼*"이라 사실상 불가능 → **모양만 맞으면 성립**(동일 심볼 불요)으로 완화하거나 와일드/홀드/강제상향으로만 닿는 **빌드 보상**으로 재배치. 잭팟이 평생 안 터지는 슬롯은 금지.

### 3.7 난이도 — 다축 (HP 스펀지 금지)

| 축 | 변경 |
|---|---|
| 적 HP | 유지(완만화) |
| 적 공격력 | **스케일 추가** (`EncounterScaling`에 DamageScale) |
| 적 행동 | 섹션 진행 시 상태이상/방어/소환 해금 |
| 군집 | 섹션 2+부터 다중 적(기존 다중 로스터/진형/타겟 회수) |
| 엘리트/보스 | 고유 기믹(어픽스/페이즈) |

가드레일: *좋은 턴 = 일반 적 처치 / 갓 턴 = 원샷.* ×Mult는 Rare+로만 게이트해 초반 폭주 방지.

### 3.8 보상 케이던스 — 전투마다 드래프트

"8연속 무선택" 제거. **모든 전투 승리 후 3택1.**
- 일반: [심볼 추가 / 마이너 유물 / 회복+재스핀] 3장 중 1택.
- 엘리트·보스: **유물 3택1**(레어+ 포함) + 옵션 "심볼 1개 정화(제거)".

### 3.9 구현 매핑 (기존 코드 재배선)

| 변경 | 위치 |
|---|---|
| `Base`, `Mult` 필드 추가 | `SlotCalculationResult` / `SlotCombatRequest` |
| Base = Σ패턴가치, Mult=1 산출 | `SlotResultCalculator.Calculate` |
| 유물이 Chip/AddMult/MultMult **기여** 방출 | `RelicTurnResolver` + 유물 효과 모델 |
| `Final = (Base+ΣChip) × ((1+ΣAdd) × Π×Mult)` 조립 | `CombatTurnRequestBuilder.Build` |
| Base×Mult 카운트업 연출 | 슬롯 프레젠테이션 |

> "슬롯이 패키지 계산 → 전투는 소비만" 구조라 기존 seam에 그대로 올라간다.

# 아웃게임 설계

## 4. 죽음을 의미있게

### 4.1 원칙

**모든 죽음은 매번 3가지를 준다:**
1. **숫자가 올랐다** — 별조각 획득, 최고 깊이/점수 갱신.
2. **새것이 가까워졌다** — "다음 해금까지 ▮▮▮▯▯" 진행바.
3. **이야기·교훈이 남았다** — 런 요약(빌드·최고타·사인).

**해금 철학: 파워가 아니라 가능성을 판다.**
- ❌ "영구 +10% 피해" (트레드밀 — 코어를 스킵하게 만듦)
- ✅ "이제 런에서 *흡혈왕관*이 등장", "새 캐릭터", "새 시작 유물 선택지"
- 예외: 모바일 온보딩용 *아주 약한* 초반 부스트만 허용, 그 이상 금지.

### 4.2 별조각 (Stardust) — 영구 통화

```
별조각 = 도달깊이 × 1
       + 엘리트 처치 × 3
       + 보스 처치 × 8
       + 첫 도전과제 보너스(가변)
       × (1 + 0.1 × 승천레벨)
```
런 종료 시 정산. (수치는 초기값, 튜닝 대상.)

### 4.3 런 요약 화면 (가장 싸고 즉효)

죽는 즉시 표시:
- 도달 깊이 / 전투 수
- **최고 한 방** (피해 + 그때 Base × Mult)
- 보유 유물(빌드) 목록
- **사인(死因)**: 누구에게 어떻게 죽었나
- 획득 별조각 + **해금 진행바**
- 최고기록 비교 ("최고 28 / 이번 22")
- [한 번 더] [로비]

### 4.4 해금 상점 — 잠자는 67종을 콘텐츠로

별조각으로 **풀에 추가**한다. 묶음·순서(예시):
1. 안전 묶음 — Chip / +Mult 유물 (저렴) → 빌드 기초 다양화
2. 폭발 묶음 — **×Mult 코어 유물** → 갓빌드 가능성 확장
3. 상태이상 묶음 — 화상/감염/취약 유물 (전투 코어 정합 *후* 개방)
4. 전설·저주 묶음 (고가) → 하이리스크 변주
+ 캐릭터 해금, 시작 유물 선택지 확장.

### 4.5 캐릭터(아키타입) — 출시 3종

각 = 시작 가방 비율(`SlotSymbolPool` 프리셋) + 시작 유물(심볼-후크) + 패시브.

| 캐릭터 | 시작 가방 | 시작 유물 | 패시브 | 빌드 |
|---|---|---|---|---|
| **도박사** | 7 비중↑ | "7 매치 시 mult ×1.5" | 시작 재스핀 +1 | 하이리스크 ×Mult 폭딜 |
| **연금술사** | 레몬↑ | "레몬 매치 시 중독 2" | 중독 피해 +25% | 상태이상 도트 (Phase 2) |
| **수문장** | 다이아↑ | "다이아 매치 시 방어 = base" | 방어의 30% 반격 | 방어→반격 |

(1번 캐릭터 기본 제공, 2·3번은 해금. 추가 캐릭터로 확장.)

### 4.6 도전과제 → 칭호

도전과제 보상으로 칭호 획득, 로비 장착, **리더보드 표시**(`LeaderboardView` DesignationText 훅 준비됨).
예: 한 스핀 ×Mult 50 → **[한탕]** / 깊이 30 → **[심연]** / 무피해 보스 → **[철벽]** / 상태이상만으로 보스 → **[연금술사]** / 잭팟 1회 → **[777]**.

### 4.7 도감(Codex)

조우한 유물·심볼·몬스터 기록, 완성도 보상(별조각/칭호). 수집 동기.

### 4.8 승천(Ascension) 1~20

첫 보스 클리어 후 해금. 각 단계 = 모디파이어 1개 추가(적 HP↑/공격↑/회복↓/저주율↑/시작 재스핀↓/×Mult 등장↓ …) + **점수 배율 +0.1/레벨**. 무한 롱테일 + 리더보드 공정성(같은 승천 내 경쟁).

### 4.9 리더보드 & 일일/주간 도전

- **리더보드**: 주간 점수 + 승천 등급별 + 일일 시드별. (`WeeklyRankingSchedule` 존재 / 온라인 랭킹은 **백엔드 필요 — 별도 인프라**)
- **일일/주간 도전**: 고정 `RunSeed`(날짜 해시)로 전원 동일 보드, 짧은 목표. 소량 별조각 보상 → **매일 들어올 이유.**

### 4.10 로비 허브

현재 단일 Play → `플레이(캐릭터 선택/이어하기) · 해금상점 · 도전과제·칭호 · 도감 · 리더보드 · 일일도전` 메뉴 허브로 확장.

## 5. 모드 / 진행 플로우

```
[타이틀] → [로비 허브] ←──── 런 종료(별조각 정산·런요약·도감 갱신)
              ↓ (캐릭터 선택 → 새 런 / 이어하기)
          [무한 런]
            섹션1(전투1~10) → 보스
            섹션2(11~20)    → 다중 적·상태이상 적, 보스
            섹션3+          → 어픽스/페이즈 보스
            각 전투: 스핀 → (텔레그래프 결정) → Base×Mult → 전투 → 승리 드래프트(3택1)
          패배 → 런요약 → 로비
```
승천·일일도전은 위에 모디파이어/시드를 얹는 변주.

| **Phase 3** 롱테일 | 도전과제·칭호·도감 + 승천 + 리더보드(백엔드) + 일일도전 | 4~7주 | **2.5~4.5주** |
| | **합계** | 3~5개월 | **~2~3개월** |

**AI가 줄이는 것**: 수식 이식·유물 데이터 작성·로직·테스트·리팩터링.
**AI가 못 줄이는 것**: 재미/밸런스 찾기(플레이테스트), 연출 손맛, Unity 에디터 작업, 아트/SFX, 백엔드, 검수.
> 병목은 "코드 타이핑"이 아니라 **사람의 검수·플레이테스트·튜닝 대역폭**. (이 프로젝트엔 이미 "Unity 미실행 → 미검증" 패턴이 있음 — 생성 ≠ 완성.)

## 8. KPI / 검증

- 한 판에 **"터졌다" 순간 ≥ 1회** 체감 (정성).
- 후반 갓빌드 Mult **×10 도달 가능**, 초반 ×Mult 폭주 없음.
- 전투당 *의미 있는 결정* ≥ 1 (텔레그래프 대응).
- 첫 런 후 *해금 1개↑ 획득률* ≥ 90%.
- (메타 진입 후) 일일 도전 **재방문율** — 리텐션 핵심 지표.

## 9. 결론 & 첫 삽

> **Base × Mult 곱셈 + 심볼-후크 유물 + 폭발 연출** = "재미"의 8할.
> **모든 죽음이 별조각·해금·이야기를 남김** = "또 할 의미".
> 순서는 **재미 먼저, 메타 나중.**

**첫 삽 = Phase 0 (2~3일).** "한 스핀에 터지고, 한 번 더 돌리고 싶다"가 나오면 게임은 산다. 그 확신이 선 뒤에 본 개발(약 2~3개월)에 들어간다.
