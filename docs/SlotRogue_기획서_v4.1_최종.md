# SlotRogue 리디자인 기획서 v4.1 (최종 · 전투 유지형)

> 이 문서가 **현행 최종본**이다. 이전 v1·v2·v3은 논의 과정 기록으로만 둔다.
> v4.1 = 사용자 확정 구조 + 리뷰 보완 2가지(방어 niche, 재스핀 완충) + 튜닝 노트 반영.

---

## 0. 한 줄 요약

**제한된 스핀 안에 몬스터를 처치하고, 몬스터의 공격·상태이상을 버티면서, 라운드 성공 보상(제안)과 상점 부적으로 다음 라운드를 더 빠르게 클리어하는 짧은 슬롯 로그라이트 점수런 게임.**

---

## 1. 기획 방향

기존 게임의 핵심은 **유지**한다: 3×5 슬롯 / 몬스터 전투 / 플레이어 HP / 적 공격 / 상태이상(속성) / 유물 시너지 / 랭킹 / 매판 처음부터.

여기에 클로버핏식 구조를 **얹는다**: 라운드 목표 + 제한 스핀 + 빠른 처치 보상 + 라운드 중 상점 + 성공 보상(제안) + 부적 구매·보유 제한 + 라운드 성공 시 상점 초기화.

> 전투는 유지한다. 다만 흐름을 **"제한된 스핀 안에 몬스터를 처치하는 구조"** 로 재정리한다.

### ★ 설계 시점에 못박을 2가지 (리뷰 보완)

밸런스로 미루면 안 되고 *구조에 박아야* 하는 항목:

1. **방어가 살 자리를 보장한다.** "빨리 죽이면 보상↑ + 피해↓"라 공격이 이중 보상을 받는다. 방치하면 방어/회복 부적은 *사면 손해인 함정*이 된다. → 일부 몬스터는 *러시 불가*하게 설계하고(§16), 방어에 자체 경제 보상을 준다(§17.3).
2. **하드 제한의 RNG 억울함을 완충한다.** 제한 스핀 + 순수 가방 RNG + 재추첨 0 = "내 잘못 아닌데 짐". → 재스핀을 *기본 기능이 아니라* 부적/제안으로만 제공(§6, §17.4). 기본 전투는 단순하게 두되 원하는 사람만 RNG 보험을 빌드에 넣게.

## 2. 핵심 루프

```text
런 시작 → 라운드 시작 → 몬스터 등장
↓
제한 스핀 안에 처치 도전
  스핀 → 슬롯 피해 계산 → 몬스터 HP 감소
       → 몬스터 생존 시 공격/상태이상 → 플레이어 HP 감소
       → (스핀 사이) 상점 이용 가능
↓
처치 성공 → 남은 스핀 수에 따라 코인 보상 증가
↓
성공 보상 제안 3택1 → 상점 초기화 → 다음 라운드
↓
플레이어 HP 0 또는 라운드 실패 → 런 종료 → 점수 정산 / 랭킹 등록
```

---

## 3. 라운드 구조

각 라운드 = 몬스터 1마리(또는 그룹). 목표는 단순: **제한 스핀 안에 처치.**

```text
라운드 1 / 슬라임 / HP 120 / 제한 스핀 5
5스핀 안에 처치 → 성공, 못하면 실패.
```

몬스터는 단순 HP 스펀지가 아니다 — **살아있는 동안 플레이어를 공격**한다.

```text
스핀 1회 → 슬롯 피해 → 몬스터 HP 감소
        → 생존 시 몬스터 공격/상태이상 → 플레이어 HP 감소 → 다음 스핀
```

즉 목표 피해만 맞추는 게 아니라, **공격·상태이상을 버티며** 제한 스핀 안에 처치해야 한다.

---

## 4. 제한 스핀 (밸런스 변수)

기본 5회, 테스트 범위 3~7. 3은 첫 기준값으로 너무 빡세므로 5로 시작.

| 라운드 | 제한 스핀 | 의도 |
|---|---:|---|
| 1 | 5 | 기본 이해 |
| 2 | 5 | 첫 상점 전 압박 |
| 3 | 5 | 유물 시너지 확인 |
| 4 | 6 | 적 상태이상 대응 |
| 5 | 6 | 보스/고점 확인 |

빠른 적 = 제한↓, 탱커 적 = 제한↑ 대신 공격 강함 식으로 차별화 가능.

---

## 5. 실패 조건 (이원화)

```text
1. 제한 스핀 안에 처치 실패
2. 플레이어 HP 0
```

빨리 못 잡으면 더 맞고, 상태이상이 누적되고, 결국 죽는다. 이 구조로 **기존 전투 시스템이 유지**된다.

---

## 6. 적 공격과 상태이상

몬스터는 스핀 이후 생존 시 행동한다. 예: 기본 공격 / 방어막 / 화상 / 감염 / 슬롯 잠금 / 심볼 약화 / 다음 스핀 피해 감소.

상태이상은 *선택을 늘리는 게 아니라* **다음 스핀의 조건을 바꿔** "빨리 잡을 이유"를 강화한다.

```text
화상     : 스핀마다 피해. 느릴수록 위험.
감염     : 라운드가 길수록 추가 피해.
동결     : 다음 스핀 일부 칸 비활성.
슬롯 잠금 : 특정 칸/열이 다음 스핀에 고정.
약화     : 다음 스핀 Mult 감소.
```

**(보완) 재스핀 완충:** 하드 제한 + 위 방해 효과 + RNG 꽝이 겹치면 억울한 패배가 난다. 이를 막기 위해 **재스핀/홀드는 부적·제안으로 획득**하는 옵트인 자원으로 둔다(§17.4). 기본 전투엔 재스핀 버튼을 상시 노출하지 않는다.

---

## 7. 빠른 처치 보상 (연속식)

처치 시 코인 획득. 적은 스핀일수록 보상↑. **절벽형이 아니라 연속식.**

```text
최종 코인 = 기본 보상 × (1 + 남은 스핀 수 × 0.15)
```

예 (기본 40, 제한 5): 1스핀(남4)=64 / 2스핀(남3)=58 / 3스핀(남2)=52 / 4스핀(남1)=46 / 5스핀(남0)=40.

플레이어 감정 목표: "깨긴 깼는데 느려서 보상이 적다 → 더 빨리 잡으려면 시너지가 필요 → 상점에서 뭘 사지?"

**(튜닝 노트)**
- 계수 0.15(최대 ×1.6)는 *밋밋할 수 있음*. 빠른 처치 압박이 약하면 0.2~0.25로 상향 테스트.
- 초반엔 빌드가 없어 스핀을 못 아낌 → 속도 보너스 부트스트랩이 느림. 초기 몇 라운드 기본 보상을 후하게 줘 시동을 건다.

## 8. 코인 획득

코인 = 상점 부적 구매 재화. 획득 요소: 처치 기본 보상 / 빠른 처치 / 초과 피해 / 유물 효과 / 특정 심볼 / 상태이상 보너스.

**결과창에 내역을 반드시 표시** → "내가 고른 제안·부적 덕에 돈을 더 벌었다"를 체감.

```text
기본 처치 보상   +40
빠른 처치 보너스 +18
초과 피해 보너스 +7
레몬 상인        +12
다이아 광산      +10
총 획득 코인     +87
```

## 9. 상점

부적 구매 장소. 라운드 중간 방문 가능.

```text
상점은 스핀 사이에 방문 가능.
방문 자체는 스핀을 소모하지 않음.
구매·리롤은 코인을 소모.
라운드 성공 시 상품 초기화.
```

플레이어 고민: "지금 코인 써서 이번 라운드를 안전하게? / 아껴서 다음에 비싼 부적? / 리롤해서 빌드에 맞는 부적 찾기?"

## 10. 부적

상점에서 코인으로 구매하는 **이번 런 전용 빌드 파츠**(시너지의 핵심). 무료로 주지 않는다. 무료 보상은 부적이 아니라 **제안**(§12).

예: 레몬 상인(레몬 매치 시 코인+3) / 체리 증폭기(체리 매치 시 Mult+0.5) / 다이아 광산(다이아 등장 시 코인+5) / 세븐 과충전(세븐 포함 족보 Mult×2) / 패턴 반복기(가로 패턴 한 번 더 발동).

---

## 11. 부적 보유 제한

```text
기본 슬롯 5 / 최대 확장 8
```

가득 찬 상태에서 새 부적 구매 시 기존 하나를 버려야 한다.

목적: 좋은 것 다 사기 방지 / 빌드 방향 강제 / 상점에서 진짜 고민 / 후반 교체 판단.

## 12. 성공 보상: 제안 (3택1)

라운드 성공 시 제안 3개 중 1택. 제안은 부적이 아니라 **이번 런의 규칙·경제·심볼 구조·상점 구조를 바꾸는 보상**.

```text
레몬 보급      레몬 무늬 수 +1
체리 보급      체리 무늬 수 +1
상점 확장      상점 상품 +1
상점 할인      다음 상점 가격 -20%
부적 슬롯 확장 부적 최대 보유 +1
빠른 처치 보너스 남은 스핀 보상 계수 +0.05
고위험 계약    다음 라운드 HP +30% / 성공 코인 +70%
안전 계약      다음 라운드 HP -20% / 성공 코인 -30%
추가 기회      다음 라운드 제한 스핀 +1
```

역할: 이번 판 방향 전환 / 부적과 시너지 / 상점 판단에 영향 / 매판 다른 전개.

---

## 13. 제안 × 부적 시너지 (핵심 재미)

- 레몬 보급 + 레몬 상인 → 레몬이 자주 나오고, 나올수록 돈을 번다(경제 빌드).
- 부적 슬롯 확장 + 저가 부적 다수 → 안정 경제 빌드.
- 고위험 계약 + 세븐 과충전 → 위험하지만 한 방 빌드로 큰 보상.

감정 목표: "방금 고른 제안 때문에 이 부적이 좋아졌다 / 이 부적 샀으니 다음 제안은 이쪽."

---

## 14. 심볼 설계

심볼은 **중립**. 고정 직업 없음. 제안·부적이 *이번 런에서* 의미를 만든다. (레몬이 경제 핵심이 되기도, 세븐이 한 방 핵심이 되기도.)

단 **UI에서 현재 심볼의 의미를 보여준다:**
```text
레몬 — 현재 효과
- 레몬 무늬 수 +1
- 레몬 매치 시 코인 +3
- 레몬 매치 시 Base +2
```

---

## 15. 피해 계산 — Base × Mult

```text
Base = 족보 점수 + 부적 Base 보정
Mult = 1 + 부적 Add-Mult 보정
최종 피해 = Base × Mult × 특수 배율(×Mult 부적)
```

예: Base 12 × Mult 3.5 × 세븐 과충전 2.0 = **84 DAMAGE**.

**연산 순서(구현자용 못박기):** ① Base 합산(족보+Chip) → ② Mult = (1 + Σ Add-Mult) → ③ 특수 배율 = Π(×Mult 부적) → ④ 최종 = Base × Mult × 특수배율. (= Chip / +Mult / ×Mult 3버킷.)

**연출 필수:** 족보 점등 → Base 카운트업 → 부적 효과 순차 점등 → Mult 상승 → 최종 피해 폭발. *이 연출이 없으면 시너지 재미가 안 보인다.*

**(튜닝 노트)** §3 예시(HP120/5스핀=24딜/스핀)는 현재 코드 Base값(가로3=3)보다 훨씬 큼 → **Base 전역 스케일을 크게 올려야** 함(순수 수치 조정).

## 16. 몬스터 설계

복잡한 AI 불필요. 최소 다음 중 하나는 보유: HP / 공격력 / 공격 주기 / 상태이상 / 보상 특성.

| 몬스터 | 특성 |
|---|---|
| 슬라임 | HP 낮음, 기본 공격만 |
| 도둑 | 빠르게 처치 시 추가 코인 |
| 골렘 | HP 높음, 공격 느리지만 강함 |
| 마녀 | 약한 공격 + 화상/감염 |
| 잠금술사 | 슬롯 잠금 상태이상 |
| 기사 | 방어막 생성, 빨리 못 잡으면 처치 지연 |

목적: 라운드 목표 / 플레이어 HP·상태이상을 의미있게 / 빠른 처치 압박 / 부적 구매 이유.

**(보완) 방어 niche 보장:** 골렘·기사·마녀처럼 **러시가 불가능/비효율적인 적**을 반드시 포함해, "무조건 빨리 죽이기"가 항상 정답이 되지 않게 한다. 이 적들에서 방어/회복 부적이 *보험*으로 가치를 갖는다.

## 17. 상점 상품 분류

### 17.1 경제 부적 (코인↑)
레몬 상인(레몬 매치 코인+3) / 다이아 광산(다이아 등장 코인+5) / 황금 레버(스핀마다 코인+3).

### 17.2 피해 부적 (처치 속도↑)
체리 증폭기(체리 매치 Mult+0.5) / 세븐 과충전(세븐 족보 Mult×2) / 패턴 반복기(가로 한 번 더).

### 17.3 방어/회복 부적 (생존)
보호 부적(라운드 시작 보호막+10) / 회복 부적(처치 시 HP+5) / 정화 부적(라운드 시작 상태이상 1개 제거).
**(보완) 방어 경제 보상:** 공격 이중보상에 맞서려면 방어에도 당근이 필요 → 예 "보호막을 남긴 채 처치 시 코인 보너스", "받은 피해 0인 라운드 보너스" 같은 *방어 플레이의 경제 인센티브*를 1~2개 넣는다.

### 17.4 상점/운영 부적 (경제 운영)
할인권(가격-15%) / 리롤 쿠폰(리롤-50%) / 진열대 확장(상품+1).
**(보완) RNG 완충 부적:** "리롤 부적: 라운드당 슬롯 1회 재추첨", "홀드 부적: 칸 1개 잠그고 재스핀" 등 **재스핀/홀드를 옵트인**으로 제공(§6).

## 18. 라운드 중 상점 판단 예시

```text
라운드 3 / 몬스터 HP 180 / 제한 스핀 5
현재 누적 피해 95 / 남은 스핀 3 / 보유 코인 55 / 플레이어 HP 34

상점: 체리 증폭기 45 / 회복 부적 40 / 세븐 과충전 90

1. 체리 증폭기  → 이번 라운드 처치 확률↑, 코인 대부분 소모
2. 회복 부적    → 장기 생존↑, 처치 속도 그대로
3. 안 삼        → 세븐 과충전 위해 저축, 이번 라운드 실패 위험↑
```

이 판단이 핵심 플레이.

## 20. 랭킹

기존 시스템 활용. 단순 클리어가 아니라 플레이 내용 반영.

점수 요소: 도달 라운드 / 총 획득 코인 / 최고 한 방 피해 / 최고 Mult / 빠른 처치 횟수 / 보스 처치 / 남은 HP / 보유 부적 가치.

**랭킹이 리텐션을 담당하므로 MVP에 별도 메타 해금은 넣지 않는다.**

---

## 21. MVP 범위

**필수 구현:** 라운드 목표 / 제한 스핀 / 플레이어 HP / 몬스터 공격 / 상태이상 최소 1~2종 / 코인 획득 / 빠른 처치 보상 / 스핀 사이 상점 / 부적 구매 / 부적 보유 제한 / 성공 제안 3택1 / 성공 시 상점 초기화 / Base×Mult 연출 / 최종 점수 정산 / 랭킹 연결 / **재스핀 부적 최소 1종 / 방어 경제 부적 최소 1종 + 러시 불가 몬스터 1종**(보완).

**MVP 수치:**
```text
라운드 수 5 / 기본 제한 스핀 5 / 기본 부적 슬롯 5
상점 상품 4 / 제안 선택지 3
초기 부적 15 / 초기 제안 10 / 상태이상 1~2 / 몬스터 4 + 보스 1
```

---

## 22. MVP 부적 예시

```text
레몬 상인     레몬 매치 시 코인 +3
체리 상인     체리 매치 시 코인 +3
다이아 광산   다이아 등장 시 코인 +5
체리 증폭기   체리 매치 시 Mult +0.5
레몬 증폭기   레몬 매치 시 Base +3
세븐 과충전   세븐 포함 족보 Mult ×2
패턴 반복기   가로 패턴 한 번 더 발동
보호 부적     라운드 시작 시 보호막 +10
회복 부적     몬스터 처치 시 HP +5
상점 할인권   상점 가격 -15%
리롤 쿠폰     상점 리롤 비용 -50%
정화 부적     라운드 시작 시 상태이상 1개 제거
리롤 부적     (보완) 라운드당 슬롯 1회 재추첨
철벽 계약     (보완) 받은 피해 0인 라운드 처치 시 코인 +30
```

## 23. MVP 제안 예시

```text
레몬 보급 / 체리 보급 / 다이아 보급   해당 무늬 수 +1
상점 확장      상점 상품 +1
상점 할인      다음 상점 가격 -20%
부적 슬롯 확장 부적 최대 보유 +1
빠른 처치 보너스 남은 스핀 보상 계수 +0.05
고위험 계약    다음 라운드 HP +30% / 성공 코인 +70%
안전 계약      다음 라운드 HP -20% / 성공 코인 -30%
추가 기회      다음 라운드 제한 스핀 +1
```

## 24. MVP에서 제외

캐릭터/성급/펫/장비 / 영구 공격력 강화 / 유물 영구 해금 / 도감 / 칭호 / 일일·주간 도전 / 복잡한 보스 페이즈 / 상태이상 7종 전체.

이유: 지금 목표는 **전투 유지형 클로버핏 루프가 재밌는지** 검증. 반복 동기는 **랭킹**으로 충분.

---

## 25. 핵심 재미

전투 중 버튼 연타가 아니라 **판단**에서 나온다:
- 이번 라운드 상점을 들를까 / 코인을 지금 쓸까 아낄까 / 어떤 제안으로 다음 라운드를 바꿀까 / 어떤 부적으로 시너지를 만들까 / 슬롯 부족할 때 무엇을 버릴까 / 빠른 처치를 노릴까 안전하게 갈까.

## 26. 구현 메모 (기존 코드 매핑)

기존 자산을 *재배선*한다(신규 엔진 없음).

| 항목 | 위치 | 변경 |
|---|---|---|
| 피해 `×2` → Base×Mult×특수 | `SlotResultCalculator` / `CombatTurnRequestBuilder` | 핵심 |
| 부적이 Chip/Add-Mult/×Mult 기여 방출 | `RelicTurnResolver` + 유물 효과 모델 | 데이터+로직 |
| 제한 스핀 카운터 + 빠른 처치 보상 | 전투/라운드 진행부 | 신규(작음) |
| 코인 + 상점 + 부적 보유 제한 | 신규 화면 1 + 진행부 | 신규(가장 큼, 단 유물보상 흐름 대체) |
| 제안 3택1 | 기존 보상 UI 재사용 | 교체 |
| 전투/HP/적 공격/상태이상/LockSlot | `BattleSystem`, `StatusEffect*`, 적 플래너 | **유지**(수치 재튜닝) |
| 랭킹 | 기존 주간 랭킹 | 점수만 연결 |

## 27. 결론

순수 점수런으로 피벗하지 않는다. **전투는 유지하고**, 클로버핏식 경제·상점·제안 구조를 얹어 짧고 반복 가능한 슬롯 로그라이트로 만든다.
