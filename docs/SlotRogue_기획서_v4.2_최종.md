# SlotRogue 리디자인 기획서 v4.2 (최종 · 무한 웨이브 · 전투 유지형 · 칩 경제)

> 이 문서가 **현행 최종본**이다. v1~v4.1은 과정 기록으로만 둔다.
> 이번 확정 사항: ①리롤=칩 소모 ②광고=상점 소액 칩 추가(런당 2회) ③홀드/재스핀=부적이며 별도 기획 ④심볼 6종=체리·레몬·다이아·세븐·클로버·종.

---

## 0. 한 줄 요약

**슬롯을 돌려 몬스터를 처치하고, 빨리 잡을수록 더 많은 칩을 얻어 상점에서 부적을 사며, 제안과 부적 시너지로 빌드를 만들고, 죽을 때까지 무한 웨이브를 진행하는 슬롯 로그라이트 랭킹 게임.**

---

## 1. 핵심 정체성

기존 구조 **유지**: 3×5 슬롯 / 몬스터 전투 / 플레이어 HP / 적 공격 / 상태이상 / 부적 시너지 / 랭킹 / 매판 처음부터.

클로버핏식 구조를 **얹는다**: 제한 스핀 안에 처치 / 상점 부적 구매 / 제안 3택1로 런 규칙 변경 / 부적 보유 제한 / 죽을 때까지 무한 진행.

> 전투는 유지한다. 다만 흐름을 **"제한 스핀 안에 몬스터를 처치하는 구조"** 로 재정리한다.

### ★ 설계 시점에 못박을 것 (밸런스로 미루지 말 것)

1. **방어가 살 자리를 보장한다.** "빨리 죽이면 칩↑ + 피해↓"라 공격이 이중 보상 → 방치하면 방어/회복 부적은 함정이 된다. → 러시 불가 몬스터(§16) + 방어 자체 칩 보상(§17.3).
2. **무한모드의 발산 곡선.** 빌드 성장 slope vs 구간 난이도 slope를 *실력/빌드에 따라 죽는 깊이가 갈리게* 튜닝. 빌드가 이기면 안 죽어 랭킹 불가, 난이도가 이기면 매번 같은 벽 → 둘 다 실패.
3. **제안 ↔ 부적 경계 유지.** 제안 = 런 전체 규칙/확률(무료·영구), 부적 = 슬롯 먹는 시너지 파츠(칩 구매). 제안이 raw Base/Mult를 주면 부적과 경쟁해 슬롯 제한 결정이 희석된다.

---

## 2. 용어

- **런** = 시작부터 죽을 때까지 한 판. 최종 구조는 **무한모드**(클리어형 아님).
- **웨이브** = 몬스터 하나(또는 그룹)를 제한 스핀 안에 잡는 단위.
- **스핀** = 슬롯 1회 = 전투 1행동.
- **칩** = 이번 런용 소단위 재화(부적 구매). 큰 골드가 아니라 카지노식 작은 단위.
- **제안** = 웨이브 성공 후 3택1. 부적이 아니라 런의 규칙·확률·경제·전투 조건을 바꾸는 선택.
- **부적** = 상점에서 칩으로 사는 이번 런 전용 빌드 파츠.

---

## 3. 게임 흐름

```text
런 시작 → 웨이브1 몬스터 등장
↓
제한 스핀 안에 처치 도전
  스핀 → 슬롯 피해 계산 + 칩 획득 → 몬스터 HP 감소
       → 생존 시 공격/상태이상 → 플레이어 HP 감소
       → (스핀 사이) 상점 이용 가능
↓
처치 성공 → 남은 스핀 수만큼 칩 보너스
↓
제안 3택1 → 상점 갱신 → 다음 웨이브
↓
HP 0 또는 제한 스핀 내 처치 실패 → 런 종료 → 점수 정산 / 랭킹 등록
```

## 4. 무한 웨이브 & 구간

5웨이브에서 끝나지 않는다. **죽을 때까지** 계속. (5웨이브 = Phase 0 검증 구간일 뿐.)

난이도는 **구간 단위**로 상승:

```text
1구간 1~5 / 2구간 6~10 / 3구간 11~15 / 4구간 16~20 / 이후 계속
```

구간↑ 시 강화: 몬스터 HP↑ / 공격력↑ / 상태이상 등장 / 방어막 적 / 상점 고등급 부적 / 위험 제안.
> 기존 `WaveSchedule`이 이미 구간(themeSection)·등급을 굴리므로 재활용한다.

---

## 5. 웨이브 & 실패 조건

각 웨이브 목표 = **제한 스핀 안에 처치.** 몬스터는 단순 점수판이 아니라 생존 시 공격/상태이상을 쓴다.

**런 종료(실패) 조건 2가지:**
```text
1. 제한 스핀 안에 처치 실패
2. 플레이어 HP 0
```
빨리 못 잡으면 더 맞고, 상태이상 누적되고, 죽는다 → 기존 전투 시스템이 살아있다.

---

## 6. 제한 스핀

웨이브마다 다를 수 있음. 기본 5, 테스트 범위 3~7(3은 첫 기준값으론 빡세니 5로 시작).
예: 일반 5 / 빠른 적 4 / 탱커 6 / 보스급 6~7.

**(별도 기획)** 홀드·재스핀(슬롯 보드 재추첨) = **부적으로 제공하는 옵트인 RNG 완충**. 하드 제한의 억울한 RNG 패배를 막는 장치이며, **세부 사양은 별도 문서에서 설계.** 기본 전투엔 재스핀 버튼을 상시 노출하지 않는다.

---

## 7. 빠른 처치 보상 (소단위 칩)

빨리 잡을수록 칩↑, 단 소단위로.

```text
기본 처치 보상 6칩 + 남은 스핀 1개당 +1칩
```
예(제한 5, 2스핀 처치): 6 + 3 = **9칩**.

목적: 빨리 잡고 싶게 / 빌드 강해짐 체감 / 부적 살 재화.
**(튜닝)** 압박이 약하면 스핀당 보너스를 +2로. 초반 부트스트랩이 느리면 초기 기본 보상을 후하게.

---

## 8. 칩 획득

한 번에 많이가 아니라 **작게 자주**. 경로: 스핀 결과 / 처치 / 빠른 처치 / 초과 피해 / 부적 효과 / 제안 효과.

**결과창에 내역 표시** → "내가 고른 제안·부적 덕에 칩을 더 벌었다"를 체감.

```text
스핀 기본 보상   +1
다이아 보너스    +1
레몬 상인        +2
처치 보상        +6
빠른 처치 보너스 +3
총 획득          +13칩
```

---

## 9. 상점

부적 구매처. 스핀 사이 방문 가능, 방문 자체 무료. 웨이브 성공 시 자동 갱신, 상품 랜덤, 부적은 칩으로 구매.

고민: 지금 싼 부적 vs 모아서 비싼 부적 / 빌드에 맞는 게 없으면 리롤 / 슬롯 가득 찼는데 갈아탈까.

---

## 10. 리롤 & 광고 정책 (확정)

- **상점 리롤 = 칩 소모.** (게임 내 경제 결정으로 유지.)
- **제안 리롤 = 칩 소모.** (동일.)
- **광고 = 상점에서 소액 칩 추가, 런당 2회.** 리롤도 인런 파워도 아니다 → 점수에 거의 영향 없어 **랭킹 공정성 유지.**
  - 칩 추가량은 *작게* 유지(예: +3~5칩). 한 방 빌드를 살 수준이면 안 됨.
  - MVP 내부 테스트는 광고 SDK 없이 버튼 누르면 즉시 칩 지급으로 검증.

> 원칙: 광고는 *편의/소액*만 판다. **인런 파워(리롤·재스핀)는 절대 광고에 걸지 않는다.**

## 11. 부적

상점에서 칩으로 사는 이번 런 전용 빌드 파츠(시너지 핵심). 무료로 안 줌(무료 보상은 제안).

예시:
```text
레몬 상인     레몬 매치 시 칩 +1
체리 상인     체리 매치 시 칩 +1
다이아 광산   다이아 등장 시 칩 +1
체리 증폭기   체리 매치 시 Mult +0.5
세븐 과충전   세븐 포함 족보 Mult ×2
패턴 반복기   가로 패턴 한 번 더 발동
보호 부적     웨이브 시작 시 보호막
회복 부적     몬스터 처치 시 HP 회복
상점 할인권   상점 가격 감소
[홀드/재스핀] (별도 기획 — RNG 완충 옵트인)
```

## 12. 부적 보유 제한

```text
기본 슬롯 5 / 최대 확장 8
```
가득 찬 상태에서 새 부적 구매 시 기존 하나를 버려야 한다.

목적: 전부 사기 방지 / **초반 빌드 → 후반 빌드 갈아타기 고민**(체리 유지 vs 세븐 전환) / 후반 교체 판단.

---

## 13. 제안 (3택1) & 부적 시너지

웨이브 성공 시 제안 3개 중 1택. 제안 = 런의 규칙/확률/경제/전투 조건 변경.

제안 종류: 심볼 등장 확률 / 심볼 가치 / 패턴 가치 / 상점 구조 / 부적 슬롯 / 전투 조건 / 칩 보상 / 위험 계약.

**시너지 예:**
- 체리 확률↑ + 체리 상인 → 체리가 자주, 나올수록 칩↑(경제 빌드).
- 부적 슬롯+1 + 저가 부적 다수 → 안정 경제 빌드.
- 고위험 계약(HP+30%/보상+70%) + 세븐 과충전 → 위험한 한 방 빌드.

감정 목표: "방금 고른 제안 때문에 이 부적이 좋아졌다 / 이 부적 샀으니 다음 제안은 이쪽."

## 14. 심볼 (6종 · 중립)

**심볼 6종:** 체리 · 레몬 · 다이아 · 세븐 · 클로버 · 종.

심볼은 **중립** — 고정 역할 없음. 제안·부적이 *이번 런에서* 의미를 만든다. (한 런은 체리 경제, 다른 런은 세븐 한 방.)

**등장 가중치(현재 코드 기준):**
```text
높음(안정): 체리 · 클로버 · 종      (시작 가중 8)
낮음(고점): 레몬 · 다이아 · 세븐    (시작 가중 4)
```
> `SlotSymbolPool.Draw`가 이미 *가중치(개수) 비례 추첨*. 제안은 개수를 더하는 게 아니라 가중치를 조정한다(예: "체리 확률↑", "세븐 가치↑·확률↓").
> (참고: v4.2 초안은 레몬을 고확률로 뒀으나, 현재 코드는 레몬=저확률. 고/저 배치는 튜닝 노브 — 코드 기준으로 정리함.)

**UI는 현재 심볼의 의미를 표시:**
```text
레몬 — 현재 효과: 매치 시 칩 +2 / Base +2 …
```

## 15. 빌드 전환 아크 (핵심 재미)

- **초반:** 고확률 심볼(체리/클로버/종)로 안정적 칩 수급. 고점 낮음.
- **중반:** 색상/패턴/경제 시너지 확장.
- **후반:** 저확률 고가치(레몬/다이아/세븐) + 관련 부적·제안이 모이면 크게 터짐 → **갈아탈지 고민**.

부적 슬롯 제한(§12)이 이 전환을 *진짜 결정*으로 만든다.

## 16. 몬스터 & 방어 niche 보장

몬스터는 HP 스펀지가 아니라 공격/상태이상을 쓴다.

```text
슬라임   기본
도둑     빨리 처치 시 추가 칩
골렘     HP 높음·공격 느림 → 러시 불가
마녀     화상/감염 부여
잠금술사 슬롯 잠금
기사     방어막 → 한 방 빌드 억제
```

**(보완) 방어가 함정이 안 되게:**
- 러시 불가/비효율 적(골렘·기사·마녀)을 **반드시 포함** → 방어/회복 부적이 보험으로 가치.
- 방어 플레이에 **자체 칩 보상**: 받은 피해 0 웨이브 처치 +칩 / 보호막 남기고 처치 +칩 / 정화 후 처치 +칩.

## 17. 상점 상품 분류

```text
17.1 경제 부적 (칩↑)        레몬 상인 / 다이아 광산 / 황금 레버(스핀마다 칩+1)
17.2 피해 부적 (처치속도↑)   체리 증폭기 / 세븐 과충전 / 패턴 반복기
17.3 방어/회복 부적 (생존)   보호 부적 / 회복 부적 / 정화 부적  + 방어 칩 보상(§16)
17.4 운영 부적 (경제)        할인권 / 진열대 확장
17.5 RNG 완충 부적           홀드 / 재스핀  ← 별도 기획
```

## 18. 피해 계산 — Base × Mult

```text
Base = 족보 점수 + 부적 Base 보정
Mult = 1 + 부적 Add-Mult 보정
특수 배율 = ×Mult 부적
최종 피해 = Base × Mult × 특수 배율
```
예: Base 12 × Mult 3.5 × 세븐 과충전 2.0 = **84 DAMAGE**.

**연산 순서(못박기):** ①Base(족보+Chip) → ②Mult=(1+ΣAdd-Mult) → ③특수배율=Π(×Mult) → ④최종=Base×Mult×특수.

**연출 필수:** 족보 점등 → Base 카운트업 → 부적 효과 점등 → Mult 상승 → 최종 피해 폭발. (없으면 시너지 재미가 안 보임.)
**(튜닝)** 현재 Base값(가로3=3)은 너무 작음 → **Base 전역 스케일 상향** 필요.

## 19. 한 판 길이 & MVP 범위

평균 2~4분 / 5분 초과 위험. **Phase 0 = 5웨이브만 구현해 첫 구간 재미 검증**(무한모드 전체 아님).

**필수 구현:** 웨이브 구조 / 제한 스핀 / 몬스터 HP / 플레이어 HP / 몬스터 공격 / 상태이상 1~2종 / 칩 획득 / 빠른 처치 보상 / 스핀 사이 상점 / 부적 구매 / 부적 보유 제한 / 제안 3택1 / 상점 자동 갱신 / 칩 리롤 / 광고 소액 칩(버튼) / Base×Mult 연출 / 점수 정산 / 랭킹 연결 / **러시 불가 적 1 + 방어 칩 보상 1.**

**MVP 수치:** 웨이브 5(첫 구간) / 제한 스핀 5 / 부적 슬롯 5 / 상점 상품 4 / 제안 3 / 초기 부적 15 / 초기 제안 10 / 상태이상 1~2 / 몬스터 4+보스 1.

**후순위:** 상태이상 다수 / 광고 SDK / 메타 해금 / 복잡 보스 / 캐릭터·펫·장비 / 홀드·재스핀(별도 기획).

## 20. 랭킹 & 메타

런 종료 시 점수 정산. 요소: 도달 웨이브 / 최고 한 방 / 최고 Mult / 총 획득 칩 / 빠른 처치 횟수 / 보스 처치 / 남은 HP / 보유 부적 가치.
핵심 지표는 **최고 도달 웨이브 또는 최고 점수**로 시작.

**메타 해금은 MVP 제외** — 반복 동기는 랭킹이 담당. (추후 확장 시: 영구 파워❌ / 새 부적·제안·계약·몬스터 풀 해금⭕ — *가능성*만 판다.)

---

## 21. 핵심 재미

전투 중 복잡 조작이 아니라 **판단**:
- 이번 웨이브 상점을 들를까 / 지금 칩으로 싼 부적 vs 모아서 비싼 부적 / 제안으로 확률을 바꿀까 가치를 올릴까 / 초반 체리 빌드 유지 vs 세븐 전환 / 슬롯 부족할 때 무엇을 버릴까 / 광고 칩을 지금 쓸까 아낄까.

## 23. 결론

순수 점수런으로 피벗하지 않는다. **전투는 유지하고**, 클로버핏식 경제·상점·제안 구조를 얹어 *무한 웨이브 슬롯 로그라이트*로 만든다.

> 슬롯을 돌려 몬스터를 잡고 → 칩을 벌어 부적을 사고 → 제안으로 런 규칙을 바꾸고 → 부적 제한 속에서 빌드를 갈아타며 → 죽을 때까지 높은 웨이브와 점수를 노린다.

**다음 단계:** Phase 0(5웨이브) 프로토타입 — Base×Mult + 제한 스핀 + 부적 3~5개 + 칩/상점 최소 + 러시 불가 적 1. "빨리 잡고 싶고, 한 번 더 하고 싶은가"를 며칠 안에 확인.
