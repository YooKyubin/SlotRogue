# ADR-0014: 패배는 부활 유예 후 확정하고 유물 기여도는 명목 효과량으로 집계한다

**Supersedes**: none
**Superseded by**: none

## Context

첫 패배 직후 결과 화면을 확정하면 플레이어가 광고 부활 여부를 판단할 시간이 없고, 부활 광고를 결과 화면의 일반 행동과 함께 배치하면 입력 우선순위도 불명확하다. 또한 최종 결과에는 보유 유물 목록만 있어 각 유물이 런 동안 실제로 얼마나 작동했는지 알 수 없다.

현재 전투 유물 계산은 `RelicEffectRunner`가 턴마다 추가 피해·방어·회복량을 결정한다. 실제 전투 적용 이후에는 여러 효과가 합쳐지므로 개별 유물의 실피해를 역산하기 어렵다.

## Decision

- 첫 패배이면서 광고 부활권이 남아 있으면 최종 결과 확정을 5초간 보류한다.
- 유예 화면은 패배시킨 몬스터 초상화, 남은 시간 카운트다운, 광고 부활 버튼만 우선 표시한다.
- 5초 안에 버튼을 누르면 카운트다운을 중지하고 광고 결과를 기다린다. Rewarded 보상 callback이면 부활하고, 표시 실패 또는 보상 없는 종료면 최종 결과를 확정한다.
- 부활은 현재 전투를 새로 만들지 않는다. 몬스터의 HP·실드·상태이상·다음 행동 순서를 유지하고 플레이어 HP만 최대 HP의 절반으로 복구한 뒤 플레이어 턴부터 재개한다.
- 시간이 끝나거나 이미 부활을 사용한 런이 다시 패배하면 즉시 최종 결과 화면으로 전환한다.
- 유물 기여도는 `RelicEffectRunner`가 계산한 유물별 명목 효과량을 런 전체에 누적한다.
- 추가 피해는 `유물 추가 공격력 × 해당 턴 공격 횟수`, 방어와 회복은 계산된 추가량을 누적한다.
- 최종 결과에는 현재 보유한 모든 유물을 표시하며, 한 번도 발동하지 않은 유물도 발동 0회·기여 0으로 표시한다.

## Alternatives considered

- **광고가 준비될 때까지 무기한 대기** — 패배 흐름이 네트워크 상태에 종속되므로 제외했다.
- **결과 화면과 부활 버튼을 동시에 표시** — 결과 확정 행동과 부활 행동이 경쟁하므로 제외했다.
- **CombatEvent에서 유물별 실효 피해를 역산** — 합산 요청, 방어도, 과잉 피해 때문에 출처별 정확한 분리가 불가능해 제외했다.
- **발동한 유물만 표시** — 보유 유물 전체의 효율 비교 요구를 충족하지 못해 제외했다.

## Consequences

- 패배 화면 ViewState는 `ReviveOffer`와 `RunResult` 두 단계를 표현해야 한다.
- 광고 표시 실패와 보상 없는 종료를 SceneRoot에 알리는 종료 event가 필요하다.
- 전투 결과는 전투 중 누적된 유물 기여 snapshot을 포함해야 한다.
- 기여도 수치는 실제 HP 변화가 아니라 비교 가능한 명목 효과량이다.
