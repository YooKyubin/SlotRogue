# ADR-0018: 패배 결과에는 심볼별 족보 통계를 표시한다

**Status**: accepted
**Date**: 2026-06-19
**Supersedes**: ADR-0014의 최종 결과 유물 기여도 표시 항목
**Superseded by**: none
**Related design-docs**: [`rewarded-ads.md`](../design-docs/rewarded-ads.md), [`slot-core.md`](../design-docs/slot-core.md)

## Context

ADR-0014는 부활 유예 후 최종 결과에 보유 유물 전체의 명목 기여도를 표시하도록 정했다. 구현 결과 유물 이름과 발동량은 보이지만, 플레이어가 실제 슬롯 런에서 어떤 심볼 족보가 자주 나왔고 어떤 심볼이 공격력을 많이 만들었는지는 바로 알기 어렵다.

슬롯 기본 피해는 `SlotPatternResolver`가 만든 `심볼 기본 공격력 × 매칭 칸 수 × 족보 배율` 값을 `SlotResultCalculator`가 합산해 계산한다. 유물 추가 피해는 별도 계산 단계에서 더해지므로, 최종 결과에서 심볼을 비교하려면 기본 공격력과 유물로 인한 공격력을 합산하지 않고 분리해서 보여줘야 한다.

## Decision

패배 최종 결과 화면은 보유 유물 목록이나 유물 기여도 대신 심볼별 족보 등장 횟수, 기본 공격력, 유물 공격력, 합계를 표시한다.

- 전투 중 나온 각 `SlotPatternMatch`를 심볼별로 집계한다.
- 한 턴에 여러 족보가 나오면 기본 공격력을 각 족보의 `CalculatedValue` 비율로 나누어 심볼별 공격력에 더한다.
- 유물 추가 피해는 `RelicContributionDelta.TriggerPatternIndex`가 가리키는 족보의 심볼에 유물 공격력으로 따로 더한다.
- 런 보너스, 방어, 회복은 심볼별 공격력에 포함하지 않는다.
- 결과 화면에는 현재 고정 심볼 6종을 모두 표시하며, 한 번도 족보가 나오지 않은 심볼도 0회·0 공격력으로 표시한다.
- 기존 유물 기여도 누적 코드는 내부 분석과 후속 UI 재사용 가능성을 위해 유지하되 최종 패배 화면의 기본 표시는 심볼 통계로 대체한다.

## Alternatives considered

- **유물 기여도와 심볼 통계를 모두 표시** — 모바일 게임오버 화면이 길어지고 이번 요구의 핵심 정보가 묻혀 제외했다.
- **기본 공격력과 유물 공격력을 합산해 표시** — 심볼 자체 족보 성과와 유물 반응 성과가 섞여 어떤 빌드 요소가 기여했는지 보기 어려워 제외했다.
- **발동한 심볼만 표시** — 어떤 심볼이 아예 성과를 못 냈는지 비교하기 어려워 제외했다.

## Consequences

- `BattleFlowResult`는 유물 기여도와 별도로 심볼별 슬롯 통계 snapshot을 포함한다.
- `GameFlowSession`은 런 전체 심볼 통계를 누적하고, 최종 패배 화면은 이 요약 문자열을 사용한다.
- ADR-0014의 부활 유예와 패배 확정 흐름은 유지되지만, 최종 결과 통계 표시 항목은 이 ADR을 따른다.
