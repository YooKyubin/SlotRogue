# 속성 / 방해 전투 설계

**Status**: draft
**Last updated**: 2026-06-24

## Purpose

유물 조건 족보가 전투 속성으로 이어지는 규칙을 정의한다. 플레이어와 몬스터/엘리트는 같은 6속성 언어를 공유하고, 보스만 슬롯을 직접 건드리는 방해를 추가로 사용한다. 이 문서는 구현자가 나중에 다시 읽어도 구현 목표, 명세, 상한 규칙, 현재 코드와의 차이를 확인할 수 있는 기준 문서다.

## Source

기획 원본은 `slot_battle_status_interference_design_v6.html`이다. v6의 핵심 변경은 속성 누적 원칙 확정이다.

- 가산 효과는 누적을 허용하고 자연 감쇠로 제어한다.
- 곱연산/배율 효과는 횟수, 비율, 턴당 처리량 상한을 둔다.
- 하드 CC와 일반 몬스터의 슬롯 직접 방해는 1차 구현에서 제외한다.

## Decisions

| # | 결정 | 요약 |
|---|------|------|
| A1 | 공통 6속성 | 플레이어와 몬스터/엘리트는 `화상`, `감염`, `흡혈`, `취약`, `약화`, `가시`를 공유한다. |
| A2 | 보스 전용 방해 | 슬롯 가리기, 슬롯 잠금, 라인 약화, 저주 심볼 임시 추가는 보스 전용이다. |
| A3 | 확률 발동 금지 | 슬롯 족보 자체가 확률이므로 유물 조건을 만족하면 효과는 확정 발동한다. |
| A4 | 정산 단위 | “피해 1회”는 플레이어 스핀 1회의 합산 피해, 몬스터 공격 행동 1회다. |
| A5 | 누적 원칙 | 감염은 가산 누적을 허용하고 자연 감쇠로 제어한다. 취약·약화·흡혈은 남은 적용 횟수로 제어한다. 가시 Stack 정책은 ADR-0021의 후속 결정으로 남긴다. |
| A6 | 상태 수치 의미 | [ADR-0021](../adr/0021-status-effect-numeric-field-semantics.md)에 따라 요청 `Amount`를 상태별 `Magnitude` 또는 `StackCount`로 변환한다. |
| A7 | 하드 CC 제외 | 표식, 방어 파괴, 빙결, 교란, 기절, 완전 봉쇄, 유물 침묵, 심볼 풀 영구 훼손은 1차 핵심에서 제외한다. |

## Shared attributes

| 속성 | 플레이어 → 몬스터 | 몬스터/엘리트 → 플레이어 | 보스 확장 | 구현 메모 |
|------|-------------------|---------------------------|-----------|-----------|
| 화상 | 유물 조건 족보 발동 시 짧은 추가 피해 | 플레이어 턴 종료 시 피해 | 화상 후 강공격 예고, 화상과 슬롯 방해 조합 | 즉시 피해 + 다음 턴 1회 피해로 감염과 구분한다. |
| 감염 | 몬스터 턴 종료마다 피해, 피해 후 수치 1 감소 | 플레이어 턴 종료마다 피해, 피해 후 수치 1 감소 | 장기전 누적 압박 | 총 스택 상한 없음. 매턴 1 감쇠가 소프트캡이다. |
| 흡혈 | 직접 공격으로 준 실제 HP 피해의 20%만큼 타격별 플레이어 회복 | 몬스터가 직접 공격으로 준 실제 HP 피해의 20%만큼 타격별 몬스터 회복 | 흡혈 강타, 광폭 흡혈 | 행동 Snapshot 기준. 실제 발동한 행동만 횟수를 1 감소시킨다. |
| 취약 | 대상이 받는 다음 N번의 피해 정산 증가 | 플레이어가 받는 다음 N번의 공격 행동 피해 증가 | 취약 후 강공격 예고 | 플레이어 피해 정산은 스핀 합산 기준이다. 유물 발동 건별 소모 금지. |
| 약화 | 몬스터의 다음 공격 피해 20% 감소 | 플레이어의 다음 N번 스핀 정산 피해 20% 감소 | 슬롯 방해 전후 화력 억제 | 직접 공격에만 적용하며 행동 단위로 1회 소모한다. |
| 가시 | 플레이어가 태세를 얻고 피격 시 50% 확률로 고정 반사 피해 | 몬스터가 태세를 얻고 피격 시 50% 확률로 고정 반사 피해 | 보스 가시 장갑, 다단히트 견제 | 직접 공격 타격마다 독립 판정하며 상대 팀 턴 종료 시 제거한다. |

## Player to monster specs

| 이름 | 분류 | 트리거 | 규칙 | 주의 |
|------|------|--------|------|------|
| 화상 | 지속 피해 | 유물 조건 족보 발동 | `화상 N`: 부여 즉시 N 피해를 주고, 몬스터 턴 종료 시 N 피해를 한 번 더 준 뒤 제거한다. | 후반 다중 족보를 고려해 낮은 수치부터 시작한다. |
| 감염 | 지속 피해 | 유물 조건 족보 발동 | `감염 N`: 몬스터 턴 종료 시 N 피해를 주고, 피해 후 감염 수치가 1 감소한다. 총 스택 상한은 없다. | 감염 총피해는 `N(N+1)/2`라서 N=3부터 화상 N의 `2N`과 동률이다. 감염은 3스택 이상 지급해야 가치가 있다. |
| 흡혈 | 회복/생존 | 유물 조건 족보 피해 발생 | `흡혈 N`: 다음 N번의 행동에서 직접 공격으로 준 실제 HP 피해의 20%를 타격마다 올림 회복한다. | 방어막 흡수량은 제외하며, 행동 중 한 번 이상 발동해도 행동당 1회만 감소한다. |
| 취약 | 피해 증폭 | 유물 조건 족보 발동 | `취약 N`: 몬스터가 받는 다음 N번의 피해 정산이 25% 증가한다. 적용될 때마다 1 감소한다. | 정산 1회는 플레이어 스핀 1회의 합산 피해다. |
| 약화 | 피해 약화 | 유물 조건 족보 발동 | `약화 N`: 몬스터의 다음 직접 공격 피해가 20% 감소한다. 공격 행동 후 1 감소한다. | `ceil(기본 피해 × 0.8)`. 기본 피해가 1 이상이면 최소 1이며, 다단히트와 다중 대상에도 행동당 1회만 소모한다. |
| 가시 | 반격/생존 | 유물 조건 족보 발동 | `가시 N`: 플레이어가 직접 공격을 받을 때마다 50% 확률로 공격자에게 N 피해를 반사한다. 적 팀 턴 종료 시 제거된다. | 방어막에 전부 막히거나 해당 타격으로 사망해도 판정하며 반사 피해는 재귀 발동하지 않는다. |

## Boss-only interference

보스 방해는 스핀 후 선택이 최소 1개 있을 때 의미가 커진다. 릴 1개 홀드, 제한 리롤처럼 플레이어가 결과를 보고 선택하는 인터랙션이 없다면 슬롯 방해는 시각 연출에 가깝다.

| 이름 | 추천도 | 규칙 | 주의 |
|------|--------|------|------|
| 슬롯 가리기 | 1차 추천 | 다음 턴 슬롯 1~2칸을 가린다. 가려진 칸은 보이지 않지만 족보 판정에는 정상 포함된다. | 가린 칸을 족보에서 제외하지 않는다. |
| 슬롯 잠금 | 1차 추천 | 다음 턴 슬롯 1칸이 잠긴다. 잠긴 칸은 이전 결과를 유지하고 다시 굴러가지 않는다. | 초반 보스는 1칸만 사용한다. |
| 라인 약화 | 1차 추천 | 다음 턴 특정 가로/세로/대각 라인의 족보 피해와 유물 효과가 50% 감소한다. | 완전 무효화는 후반 보스 특수 패턴으로만 검토한다. |
| 저주 심볼 임시 추가 | 1차 추천 | 이번 전투 동안 심볼 풀에 저주 심볼 1개를 추가한다. 전투 종료 시 제거된다. | 1차 저주 심볼은 효과 없는 꽝 심볼로 시작한다. |
| 심볼 오염 | 조건부 | 다음 턴 특정 심볼이 포함된 유물 효과가 50% 감소한다. | 빌드 저격 느낌이 강하므로 후반 패턴으로만 사용한다. |
| 회복 감소 | 조건부 | 다음 N번의 회복량이 50% 감소한다. | 회복 빌드가 아니면 죽은 패턴이므로 보조 피해와 함께 사용한다. |
| 유물 약화 | 후순위 | 다음 턴 가장 최근에 발동한 유물 1개의 효과가 50% 감소한다. | 유물 발동 불가는 비추천이다. |
| 압박 | 후순위 | 다음 턴 X 이상의 피해를 주지 못하면 플레이어가 추가 피해를 받는다. | 요구 피해량 조정이 어려우므로 1차 이후로 미룬다. |

## Implementation targets

상태 수치의 계층별 의미와 상태별 `Amount` 매핑은 [ADR-0021](../adr/0021-status-effect-numeric-field-semantics.md)을 따른다. 가시 Stack 정책과 기존 `RelicDerivedHeal` 흡혈 통합 여부는 후속 논의로 남긴다.

### 전투 코어

- `StatusEffectKind`는 최소 `Burn`, `Infection`, `Lifesteal`, `Vulnerable`, `Weaken`, `Thorns`를 표현해야 한다.
- 기존 `Poison`은 v6 감염과 계약이 다르므로 `Infection`으로 교체하거나 명확히 deprecated 처리한다.
- `Freeze`는 1차 핵심에서 제외한다. 기존 debug/test 용도는 남길 수 있지만 유물/몬스터 공식 데이터에는 사용하지 않는다.
- 상태이상 훅은 턴 시작뿐 아니라 턴 종료, 피해 정산 전, 피해 정산 후, 피격 후 반응을 지원해야 한다.
- 취약/약화는 피해 개별 유물 발동이 아니라 정산 단위에 적용되어야 한다. 약화는 공격자의 outgoing modifier로 먼저 적용하고 취약은 대상의 incoming modifier로 이어서 적용한다.
- 흡혈은 타격 전후 HP Snapshot 차이로 계산한 실제 HP 피해량을 기준으로 회복한다. `EffectApplyResult.DamageDealt`의 기존 의미는 변경하지 않는다.
- 가시는 직접 공격 타격 후 흡혈 처리가 끝난 다음 50% 확률로 공격자에게 고정 반사 피해를 준다. 반사 피해는 `DamageOrigin.Reflection`으로 구분해 Modifier와 후속 반응을 적용하지 않는다.

### 유물 실행

- `RelicEffectRunner`는 조건 만족 시 상태/흡혈/가시 요청을 확정 생성한다.
- 상태 유물은 보상풀에서 `Phase1=false`로 묶여 있으므로, 전투 계약 구현 후 해당 유물의 `Phase1` 전환 범위를 별도로 정한다.
- `EnemyStatusRequirement.Any`는 화상/감염/취약/약화를 포함하고, 빙결은 포함하지 않는다.
- 상태 조회는 `RelicTurnContext.FromBattle()`에서 `Burn`, `Infection`, `Vulnerable`, `Weaken`을 구분해 전달한다.

### 몬스터/보스 데이터

- 일반 몬스터/엘리트는 슬롯 직접 방해를 사용하지 않는다.
- 몬스터 SO 효과 정의에는 상태 부여, 흡혈 공격, 가시 태세를 표현할 런타임 효과가 필요하다.
- 보스 슬롯 방해는 `LockSlotEffectDefinition`처럼 Core-safe 런타임 표현과 Slot 계층 적용 API를 분리한다.

### UI / 연출

- 상태 아이콘은 6속성 기준으로 표시한다.
- 상태 틱, 상태 만료, 가시 반사, 흡혈 회복, 취약/약화 소모는 `CombatEvent`로 구분 가능해야 한다.
- 보스 방해는 예고 UI가 필수다. 특히 취약 후 강공격, 슬롯 잠금/가리기는 적용 전 턴에 보여준다.

## Current implementation snapshot

2026-06-22 기준 현재 코드는 v6의 최종 규칙이 아니라 이전 속성 연결 작업의 중간 상태다.

| 영역 | 현재 상태 | v6 기준 차이 |
|------|-----------|--------------|
| 상태 엔진 | `StatusEffectEngine`, `StatusEffectFactory`, component 구조 존재 | 훅이 턴 시작/턴 종료/스킵 중심이라 피해 정산 전후와 피격 반응이 부족하다. |
| 상태 종류 | `Burn`, `Freeze`, `Poison` | v6 6속성 중 `Burn`만 이름이 맞고, `Poison`은 `Infection`과 타이밍/감쇠/상한이 다르다. `Freeze`는 제외 대상이다. |
| 화상 | `PeriodicDamageComponent`가 턴 시작에 피해, duration 감소로 만료 | v6은 부여 즉시 피해 + 몬스터 턴 종료 1회 피해 후 제거다. |
| 감염 | `Poison`으로 임시 매핑, 턴 시작에 stack 수만큼 피해, 5스택 하드캡, 감소 없음 | v6은 턴 종료 피해 후 수치 1 감소, 총 스택 상한 없음이다. |
| 취약/약화/흡혈/가시 | 취약·약화 modifier, 실제 HP 피해 후 흡혈, 직접 피격 후 50% 반사 및 상대 팀 턴 종료 제거까지 Core 구현 완료 | 유물 요청 생성과 상태 UI 연결이 필요하다. |
| 유물 | `RelicCatalog`에 상태 유물이 등록되어 있으나 대부분 `Phase1=false` | 전투 계약 구현 후 보상풀 활성화 필요. |
| 몬스터 행동 | Damage/Shield/Heal/LockSlot 런타임 변환 존재 | 상태 부여/흡혈/가시 태세는 아직 데이터 효과 정의와 변환이 없다. |
| 보스 방해 | `LockSlotEffectDefinition`과 `EnemyActionEffectKind.LockSlot` 표현 존재 | 실제 슬롯 잠금 적용, 슬롯 가리기, 라인 약화, 저주 심볼은 미구현이다. |

## Suggested implementation order

1. `StatusEffectKind`와 상태 조회 계약을 v6 이름으로 정리한다.
2. 화상/감염을 v6 타이밍으로 교체하고 EditMode 테스트를 갱신한다.
3. 피해 정산 전후 훅으로 취약/약화를 구현한다. 취약과 약화 구현은 완료되었다.
4. 실제 HP 피해 기반 흡혈을 구현한다. 행동 Snapshot과 Revision 보호를 유지하며 실제 발동한 행동만 횟수를 1 감소시킨다. 구현 완료.
5. 피격 후 반응 훅으로 가시와 상대 팀 턴 종료 제거를 구현한다. 구현 완료.
6. 유물 `ApplyBurn`, `ApplyInfect`, `ApplyVulnerable`, `ApplyWeak`, `GainThorns`, `Lifesteal`을 단계적으로 `Phase1`에 편입한다.
7. 몬스터/엘리트 상태 행동 데이터를 추가한다.
8. 보스 슬롯 방해는 Slot 계층의 스핀 후 선택 API가 정리된 뒤 적용한다.

## Alternatives considered

- 기존 `Poison`을 감염으로 그대로 사용: 턴 시작 피해, 5스택 하드캡, 감소 없음이 v6과 달라 거절한다.
- 취약을 유물 발동 건별로 소모: 다중 족보 턴에 효과가 첫 건에만 증발하므로 거절한다.
- 가시를 자신의 팀 턴 종료에 제거: 상대 공격 전에 사라져 발동 기회가 없으므로 거절한다.
- 일반 몬스터가 슬롯 잠금/가리기 사용: 이해 비용과 억까 감각이 커져 보스 전용으로 제한한다.
