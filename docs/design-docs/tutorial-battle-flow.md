# 튜토리얼 전투 흐름

**Status**: draft  
**Last updated**: 2026-06-23

## Purpose

최초 튜토리얼 전투에서 플레이어가 몬스터 의도, 인벤토리의 보유 유물 확인, 족보 발동, 유물 반응, 위험한 적 선처치, 몬스터 턴 후 다음 의도 갱신을 한 번의 짧은 전투 안에서 이해하게 한다.

## Runtime flow

1. `GameStart` 시작 버튼은 최초 1회 완료 플래그가 없으면 `GameFlowSession.StartTutorialRun()`으로 `RunGame` 전투에 바로 진입한다.
2. 로비 임시 버튼 `튜토리얼 시작`은 완료 플래그와 무관하게 튜토리얼 전투를 바로 시작한다.
3. 플레이어는 HP 20, 보호막 0, `훈련용 배터리` 1개를 가진다.
4. 왼쪽 몬스터는 HP 6, 첫 의도 공격 4로 시작한다.
5. 오른쪽 몬스터는 HP 8, 첫 의도 방어 3으로 시작하고, 방어 후 다음 의도는 공격으로 바뀐다.
6. 전투 시작 안내 중에는 SPIN 입력을 잠그고 인벤토리 확인을 유도한다.
7. 인벤토리를 열면 유물 탭에서 `체리 3개 이상: 추가 피해 +3`, `레몬 3개 이상: 보호막 +4`를 확인한다.
8. 인벤토리를 닫으면 첫 SPIN이 허용된다.
9. 첫 SPIN은 체리 3개와 레몬 3개 족보가 모두 나오며, 체리 족보에 `훈련용 배터리` 추가 피해 +3, 레몬 족보에 보호막 +4가 반응한다.
10. 왼쪽 몬스터는 첫 플레이어 턴에 처치되어 공격하지 않는다.
11. 몬스터 턴에는 오른쪽 몬스터만 방어 3을 실행하고 다음 의도가 공격으로 갱신된다.
12. 두 번째 SPIN은 강한 체리 5개 족보로 고정되어 오른쪽 몬스터 HP 8 + 보호막 3을 넘긴다.
13. 승리 시 튜토리얼 완료 플래그를 저장하고 완료 안내 후 로비로 복귀한다.

## Boundaries

- `TutorialSlotSpinFactory`는 튜토리얼 1번째/2번째 스핀만 고정 결과를 제공한다.
- `SlotMachineService`의 override가 `null`을 반환하면 일반 랜덤 스핀으로 돌아가므로 일반 전투 RNG에는 영향이 없다.
- `훈련용 배터리`는 `RelicCatalog` 보상풀에 넣지 않고 튜토리얼 세션 보유 유물에만 추가한다.
- 체리/레몬 자체가 공격/방어를 의미하지 않는다. 기본 피해는 족보 가치에서 나오고, 공격/방어 추가 효과는 유물이 족보에 반응해 만든다.
