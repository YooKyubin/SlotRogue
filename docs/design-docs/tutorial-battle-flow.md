# 튜토리얼 전투 흐름

**Status**: draft  
**Last updated**: 2026-06-29

## Purpose

최초 튜토리얼 전투에서 플레이어가 몬스터 의도, 인벤토리의 심볼 확률 확인, 족보 발동, 위험한 적 선처치, 몬스터 턴 후 다음 의도 갱신을 한 번의 짧은 전투 안에서 이해하게 한다.

## Runtime flow

1. `GameStart` 시작 버튼은 최초 1회 완료 플래그가 없으면 `GameFlowSession.StartTutorialRun()`으로 `RunGame` 전투에 바로 진입한다.
2. 로비 임시 버튼 `튜토리얼 시작`은 완료 플래그와 무관하게 튜토리얼 전투를 바로 시작한다.
3. 플레이어는 HP 20, 보호막 0, 보유 유물 0개로 시작한다.
4. 왼쪽 몬스터는 HP 6, 첫 의도 공격 4로 시작한다.
5. 오른쪽 몬스터는 HP 8, 첫 의도 방어 2로 시작하고, 방어 후 다음 의도는 공격으로 바뀐다.
6. 전투 시작 안내 중에는 SPIN 입력을 잠그고 인벤토리 확인을 유도한다.
7. 인벤토리를 열면 심볼 확률 탭에서 현재 심볼별 출현 확률과 가중치를 확인한다.
8. 인벤토리를 닫으면 첫 SPIN이 허용된다.
9. 첫 SPIN은 체리 3개와 레몬 3개 족보가 모두 나오며, 두 족보가 기본 피해로 전환되는 흐름을 보여준다.
10. 왼쪽 몬스터는 첫 플레이어 턴에 처치되어 공격하지 않는다.
11. 몬스터 턴에는 오른쪽 몬스터만 방어 2를 실행하고 다음 의도가 공격으로 갱신된다.
12. 두 번째 SPIN은 강한 체리 5개 족보로 고정되어 오른쪽 몬스터 HP 8 + 보호막 2를 넘긴다.
13. 승리 시 튜토리얼 완료 플래그를 저장하고 완료 안내 후 로비로 복귀한다.

## Boundaries

- `TutorialSlotSpinFactory`는 튜토리얼 1번째/2번째 스핀만 고정 결과를 제공한다.
- `SlotMachineService`의 override가 `null`을 반환하면 일반 랜덤 스핀으로 돌아가므로 일반 전투 RNG에는 영향이 없다.
- 튜토리얼도 시작 유물을 지급하지 않는다. 보유 유물 안내는 전투 중 상점 구조가 안정된 뒤 별도 튜토리얼로 분리한다.
- 체리/레몬 자체가 공격/방어를 의미하지 않는다. 기본 피해는 족보 가치에서 나오고, 방어/회복 같은 추가 효과는 유물이 족보에 반응해 만든다.
