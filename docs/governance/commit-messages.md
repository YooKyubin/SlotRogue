# 커밋 메시지 규칙

커밋 메시지는 첫 줄에 type과 한 줄 요약을 쓰고, 빈 줄 뒤에 불렛 리스트로 세부 내용을 적는다.

---

## Type

자주 쓰는 type:

- `feat`: 새 기능
- `fix`: 버그 수정
- `docs`: 문서 변경
- `refactor`: 동작 변경 없는 구조 개선
- `test`: 테스트 추가/수정
- `chore`: 빌드, 설정, 메타 파일 등 보조 작업

---

## 형식

```text
feat: <작업 내용 한줄 요약>

- <세부 작업 내용 1>
- <세부 작업 내용 2>
- <세부 작업 내용 3>
```

---

## PowerShell 필수 작성 방식

PowerShell에서 커밋 메시지를 작성할 때는 **반드시** here-string을 사용해 하나의 메시지로 전달한다. 여러 `-m` 옵션을 나누어 쓰거나 줄별로 따로 전달하면 불필요한 줄바꿈이 생기기 쉬우므로 사용하지 않는다.

```powershell
git commit -m @"
feat: <작업 내용 한줄 요약>

- <세부 작업 내용 1>
- <세부 작업 내용 2>
- <세부 작업 내용 3>
"@
```

첫 줄은 1개 커밋의 결과를 바로 읽을 수 있게 짧게 쓰고, 불렛 리스트에는 실제 변경 파일/시스템/검증 내용을 구체적으로 적는다.
